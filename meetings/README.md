# meetings — 議事録・進捗管理（例）

ミーティングの記録と進捗管理ドキュメントを置く場所。

## 構成

| ファイル | 内容 |
|---|---|
| `_template.md` | 議事録テンプレート（コピーして使う） |
| `wbs.md` | WBS（作業分解）の記入例 |
| `gantt.md` | ガントチャート（Mermaid）の記入例 |
| `YYYY-MM-DD_第N回議事録_*.pdf` | 本チームで回している正規の議事録（PDF） |

## 議事録の作り方

本チームは**PDFの正規議事録をコミットする運用**。議事録担当者が LaTeX 等で
作成した PDF をこのフォルダに置く。

正規議事録に抜けていた決定事項（例: IMU採用）は、議事録を書き直さず
[`docs/decisions/`](../docs/decisions/) に ADR（設計判断記録）として残す。

1. 担当者が議事録 PDF を作成（ファイル名規則: `YYYY-MM-DD_第N回議事録_*.pdf`）
2. このフォルダに置いてコミット
3. 補足や PDF で漏れた決定事項は ADR に記録

`_template.md` は markdown で議事録を取りたい場合の参考テンプレ（汎用）。

## 進捗管理のコツ

- **WBS** は最初のうちに作る。粒度は「1タスク = 数時間〜3日」が目安
- **ガントチャート** は WBS と連動させる。Mermaid は GitHub で自動描画される
- 毎週のミーティングで進捗確認し、WBS の「状態」列を更新する

## 不要な班は

進捗管理を別のツール（Notion, Trello, GitHub Projects 等）で行うなら、
このディレクトリごと削除してよい。
