## 3. 現状分析（なぜ肥大・崩壊したか）

`23_計画書・設計書_24G1075.tex` は 3077 行・178 KB。問題は 3 つ。

### 3-1. Chapter 2・3 が「2 版収録」になっている（最大要因）

複数人のマージ済みドラフトをさらに連結したため、同じ節が 2 回出てくる。

- **Chapter 2「システムの基本設計」**
  - Block A（行 506–816）= 音楽/Processing 班マージ版。`機能一覧/システム構成図/操作IF/状態遷移図` を
    サブシステム別（`Processing音源サブシステム`／`楽譜進行・発音情報配信`）に記述。
  - Block B（行 817–1209）= Arduino/EMA 班マージ版。同じ 4 節を `要求一覧＋FBS`／`全体ブロック図` 等で
    体系的に記述（`\label` 付き）。
- **Chapter 3「システムの詳細設計」**
  - Block A（行 1214–1872）= 音楽/Processing 班版。`部品一覧/HW/SW/処理フロー/テスト`。
  - Block B（行 1873–3056）= Arduino/EMA 班版。同 5 節（`\label` 付き）。

→ **方針: Block B（`\label` 付き・体系的）を土台にし、Block A にしか無い事実
（Processing 音色・NOTE 受信フロー・ボイス管理など）だけを畳み込んで 1 版に統合**。残りは削除。
重複行マップは付録 A。

### 3-2. 図が壊れている／欠損している

- `\includegraphics` 参照は 5 点。うち **3 点が実体不在**:
  `WBS.PNG`・`arrow.png`（アローダイアグラム）・`ganto.png`（ガントチャート）はリポジトリに無い。
  `FBS.png`／`PBS.png` も `meetings/.../PBS/FBS/` にしか無く、`計画書結合/` 直下には無い（`\graphicspath` 未設定）。
- TikZ 図が 8 点（行 616/766/896/1134/1174/1287/1781/2092）。`\resizebox` で潰しているものがあり、
  レンダリング品質・はみ出しは要実測。

### 3-3. 表の書式崩れ

`longtable` 30・`table` 11。多くが手動 `p{...\textwidth}` 指定で、列幅合計が版面を超えてオーバーフロー
している疑いが濃い。`lstlisting` 4・`verbatim` 6 もあり、コード塊がページを食っている。

## 4. 確定した方針（ユーザー確認済み）

- **図の作成方式**: Claude が `.drawio`（mxGraph XML）を直接記述 → draw.io の CLI で PNG 書き出し →
  `\includegraphics` で挿入。draw.io 専用 MCP は使わない（未導入のため）。
- **図の変換スコープ**: 欠損 3 点（WBS/アロー/ガント）と、ビルドで崩れ・はみ出しが確認できた図を
  優先で draw.io 化。正常にレンダリングできている TikZ は据え置き可。
- **設計章の粒度**: 事細かに書かず大まか。プロジェクトの内容が伝われば可。長い API 表・コード例は削減。
- 章立ては `plan_template.tex`（課題指定）に従う: 計画書／基本設計／詳細設計／生成 AI 利用。

### 4-1. draw.io 書き出し環境（未整備 — Phase 0 で構築）

現状: draw.io デスクトップ無し・`drawio` CLI 無し・Docker はインストール済だが停止中・VS Code 拡張無し。

採用する書き出し手段（Phase 0 でどちらか確立）:

- **第 1 候補**: `brew install --cask drawio` でデスクトップを入れ、同梱 CLI で書き出す:
  `/Applications/draw.io.app/Contents/MacOS/draw.io --export --format png --scale 2 -o fig/x.png fig/x.drawio`
- **第 2 候補**: Docker 起動 → ヘッドレス書き出しイメージ（`rlespinasse/drawio-desktop-headless` 等）。

インストールはユーザー操作が要る場合がある。Phase 0 で `! brew install --cask drawio` を案内する。

## 10. draw.io ワークフロー詳細

1. **作図**: Claude が `fig/<name>.drawio` を mxGraph XML として直接 Write する。
   - 文字は日本語可。フォント崩れ回避のため標準フォント指定。
   - A4 1 段に収まる縦横比（横長なら幅優先）。LaTeX 側は `width=\linewidth` 等で制御。
2. **書き出し**: §4-1 で確立した手段で PNG 化。`--scale 2`（高 DPI）推奨。`fig/<name>.png` に出力。
3. **挿入**: 本体 `.tex` で
   ```tex
   \begin{figure}[H]\centering
     \includegraphics[width=0.9\linewidth]{<name>.png}
     \caption{...}\label{fig:...}
   \end{figure}
   ```
4. **コミット**: `.drawio` と `.png` を両方 add（後から編集可能にするため XML を必ず残す）。
5. 図を直すときは `.drawio` を編集 → 再書き出し → 差し替え。`.png` を手で描き換えない。
