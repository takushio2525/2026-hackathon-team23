## FAIL 項目の分析と改善策

### node_02 の WiFi 品質問題（MOP3/MOP5/MOP9 に共通）

node_02 が MOP3（楽譜ズレ）・MOP5（受信数不足）・MOP9（13.3% パケロス）の3項目で唯一の不良個体。他4台は全て良好。

- 原因候補: アンテナ位置、SoftAP との物理距離、USB ケーブルからのノイズ、ハードウェア個体差
- 改善策: node_02 の物理配置変更、または別の Arduino UNO R4 WiFi への交換
- 見込み: node_02 改善で MOP3・MOP9 は PASS、MOP5 も改善方向

### MOP4 同期誤差のスパイク

平均 14ms・中央値 13ms で大半は目標 20ms 以内。最大 29ms は WiFi ジッタ由来のスパイク。

- 改善策: beatLookaheadMs の調整、または p95（25ms）ベースでの判定に緩和

### MOP5 通信遅延の計測限界

ahead ベースの遅延推定にはマスタクロック同期の EMA 精度が含まれるため、純粋な UDP 通信遅延（おそらく数 ms）より大きく出る。

- 改善策: 計測手法の見直し（同一 PC 上のタイムスタンプ比較、またはオシロスコープによるハードウェア計測）
- 注記: 実際の演奏体験としては同期誤差 20ms 以内で「ズレを感じない」水準を達成している（MOP4 で検証済み）
