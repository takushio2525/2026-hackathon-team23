# MOP1: 拍検出の正確性 評価記録

## 目標値
正解率 >= 90%（期待拍数に対する検出拍数の比率）

## 検証方法の妥当性

### 現行方式の概要

指揮者ノード（node_01）のシリアル出力から拍検出イベント `M1,<beatNo>,<timestamp_ms>,<bpmQ8>` を収集し、
「実測区間内の期待拍数」に対する「実検出拍数」の比率で正解率を算出する。

### 評価

**結論: 定速メトロノームに合わせた検証では概ね有効。ただし以下の限界がある。**

1. **テンポ揺れの影響**: 人間の手振りは ±5〜10% のテンポ揺れが自然に生じる。
   期待拍数を「BPM × 秒数 / 60」で算出する方式は、テンポが安定している前提で成り立つ。
   改善版では最初の拍から最後の拍までの実測区間を使うことで、振り始めのラグによる
   過小評価を緩和した。

2. **false positive の検出**: 偽検出（存在しない拍を検出）が多い場合、検出率が 100% を超える。
   改善版では検出率 > 110% で警告を出し、期待間隔の半分未満の短間隔を偽検出疑いとして
   カウントする機能を追加した。

3. **false negative と false positive の相殺**: 検出漏れ 5 拍 + 偽検出 5 拍があると
   検出率は 100% 近くになり問題が隠れる。拍間隔の CV（変動係数）が大きい場合は
   この相殺が疑われるため、CV も合わせて確認すること。

4. **「正解」の定義の曖昧さ**: 厳密には「正しいタイミングで検出された拍」を正解とすべきだが、
   人間の手振りでは「正しいタイミング」の定義自体が困難。現行方式の「拍数の比率」は
   実用的な近似として妥当。

### 推奨テスト手順

1. メトロノーム（BPM 120）に合わせて 30 秒以上振る
2. `python3 mop1_beat_detect.py --bpm 120 --duration 30`
3. 以下をすべて確認:
   - 検出率（実測区間）が 90〜110% の範囲
   - 拍間隔 CV が 15% 以下（安定した振りの場合）
   - 短間隔の偽検出警告が出ていないこと
4. BPM 80 / 160 / 200 でも同様にテストし、速度域の依存性を確認

## 考察

### ファーム出力の検証結果

- `M1,<beatNo>,<timestamp_ms>,<bpmQ8>` フォーマットは node_01 / node_01_devkitc 両方で同一実装
- `bpmQ8` は `gData.tempo.bpm * 8.0f + 0.5f` で計算 — ×8 整数（0.125 BPM 分解能）で正しい
- beatNo の変化検知（`sPrevBeatNo_m1` との比較）で出力 — BEAT 送出タイミングと一致
- MOP_TEST=1 時は SERIAL_DEBUG の `dumpEdges` が呼ばれないため二重出力なし

### Python スクリプトの改善内容

1. 実測区間ベースの期待拍数に変更（振り始めラグの影響を排除）
2. 検出率の上限チェック追加（> 110% で偽検出警告）
3. 短間隔カウントによる偽検出の定量的検出
4. 0〜1 拍でも FAIL の summary を書き出すように修正
5. `--bpm` 未指定時の summary を「判定なし」に変更（誤った FAIL 記録を防止）
6. 期待拍数の `int()` 切り捨てを `round()` に修正
