# MOP6: テンポ追従の遅延 評価記録

## 目標値
テンポ変化後 <= 2 拍以内に追従すること

## 検証方法の妥当性

### 計測方式
- 指揮者ノード（SERIAL_DEBUG=1）の `[N1 EVT BEAT]` から拍ごとの BPM を取得
- 楽器ノード（MOP_TEST=6）の `M6` 出力から毎拍の受信 BPM を取得
- PC 側タイムスタンプで両者を時系列に並べ，テンポ変化→追従を計測

### 追従遅延の定義
指揮者の BPM が拍間で 10% 以上変化した時点を「テンポ変化」と定義し，
その後，楽器側の BPM が新テンポの ±15% 以内に入るまでの時間を拍数に換算する．

### 妥当性の評価
- **10% 閾値**: EMA の漸次変化を含め，人間が知覚できるテンポ変化を検出できる水準．微小な揺らぎを誤検出しない程度に高い
- **15% 収束判定**: 目標 BPM の ±15% は，人間の聴感で「追従した」と感じる範囲
- **制約**: 指揮者側の EMA 平滑化による BPM 推定遅延もこの計測に含まれる．純粋なネットワーク遅延のみの分離は MOP4（同期精度）の計測で行う
- **データソースの併用**: M6 フォーマット（MOP_TEST=6）と SERIAL_DEBUG フォーマットの両方をパースするため，設定の組み合わせによらず計測可能
