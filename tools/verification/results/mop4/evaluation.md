# MOP4: 楽器間同期誤差 評価記録

## 目標値
最大同期誤差 <= 20 ms（同一拍に対する楽器間の発音時刻差）

## 検証方法の妥当性

### 計測原理

各楽器ノードは BEAT 受信時に `localMasterMs = millis() + offsetMs`（マスタクロック同期済み
ローカル推定時刻）をシリアル出力する。発音は `localMasterMs >= playAtMasterMs` で発火する。
`playAtMasterMs` は指揮者が決める全ノード共通の値なので、同一拍の `localMasterMs` の
楽器間最大差がそのまま発音タイミング差（= 同期誤差）になる。

旧 `analyze.py` は PC 側タイムスタンプのクラスタリングで計測していたが、USB シリアル転送
遅延（数 ms〜数十 ms、ポートごとにばらつく）が乗るため不正確だった。現行方式は楽器側で
マスタクロック基準のタイムスタンプを生成するため、USB 遅延の影響を受けない。

### SERIAL_DEBUG フォールバック

MOP_TEST=4 未設定でも `[NX EVT BEAT] no=... playAt=... ahead=...` 形式から
`localMasterMs = playAt - ahead` で逆算可能。`ahead = playAtMasterMs - localMasterMs`
であるため数学的に等価。

### 既知の制約

1. **ループジッタ**: 本計測は BEAT 受信時点の `localMasterMs` を使うが、実際の発音は
   次のループ周期で `localMasterMs >= playAtMasterMs` を満たしたときに発火する。
   ループ周期は production で 2 ms なので、最大 2 ms のジッタが計測値に含まれない。
   目標値 20 ms に対して十分小さく、判定結果への影響は無視できる。

2. **シリアルポーリング**: PC 側で複数ポートを逐次読みするため、高テンポ・多ノードの場合に
   一部の拍でデータ取りこぼしが生じ得る。同期誤差の計算精度には影響しないが、
   有効サンプル数が減少する可能性がある。

3. **headRestBeats**: 楽器ごとに演奏開始拍が異なるため、序盤の拍では一部の楽器のみが
   発音し、2 台以上の同時発音が揃わない。スクリプトは 2 台未満の拍を自動除外する。

### 妥当性の結論

マスタクロック同期済みの楽器側タイムスタンプを直接比較する方式は、USB 遅延の混入を排除
しており、計測原理として妥当である。既知の制約（ループジッタ最大 2 ms）を加味しても
20 ms の合否判定に影響しない。
