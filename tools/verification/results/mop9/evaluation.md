# MOP9: パケットロス耐性 評価記録

## 目標値
5% パケロス環境下で聴感破綻なし

## 検証方法の妥当性

### 計測方式: beatNo ベース（正）
beatNo の欠番を拍レベルの欠落として計測する。seq（パケット連番）の欠番ではない。
beatRedundancy=4（同一 BEAT を4連送）のため seq には連送分の「正常な飛び」が含まれ、
seq の欠番 ≠ 拍の欠落。beatNo の欠番のみが「楽器が拍を受け取れなかった」ことを示す。

ファーム側の `MOP_TEST=9` 出力は `isDuplicate` チェック後（新規 beatNo のみ出力）に
配置されており、beatNo ベース計測に適切なデータを提供する。

### 指揮者リセット対応
指揮者再起動で beatNo が 0 にリセットされる場合、beatNo の減少を検出して
区間分割し、各区間のロスを独立計算して集計する。

### 最大連続欠落数
単発ロスよりバースト欠落が聴感破綻に直結するため、最大連続欠落数をレポートに含める。

### 制約と課題
1. **疑似パケロス未実装**: MOP定義は「5%パケロス環境下で聴感破綻なし」を
   テスト条件として要求しているが、本スクリプトは自然環境の実測ロス率を
   計測するのみ。5% ストレステストには以下のいずれかが必要:
   - ファーム側でランダムに BEAT を破棄する仕組み
   - ネットワーク層でのパケロス注入（Linux: `tc netem loss 5%`）
   - 物理的に劣悪な無線環境の構築
2. **聴感評価は人間判断**: 「破綻なし」は聴感に基づく主観評価。
   本スクリプトのロス率・最大連続欠落数は定量的な参考指標として使用する。
3. **beatRedundancy 効果の定量化不可**: 現ファーム出力は isDuplicate 判定後の
   1行/beatNo のみ。連送4パケットのうち何個到達したかは計測できない。
