# canon_sim — test_v3 4 台輪唱の机上シミュレーション

`firmware/test_v3` の楽器ノード（node_02〜05）の楽譜進行ロジック
（輪唱サイクル窓方式・`CANON_CYCLE_BEATS=56`）を Python で忠実に再現し、
拍番号 1〜120 の発音タイミング表を生成して輪唱の成立を数値検証するスクリプト。

2026-06-10 の実機テストで「node_03/04 が輪唱に聞こえない」問題が出た際の
机上検証として作成。**結論: 現行ファームのロジックは正しい**（下記 20 項目全 PASS）。

## 実行

```bash
python3 tools/canon_sim/canon_sim.py
```

依存なし（標準ライブラリのみ）。全アサーション PASS なら exit 0。

## 再現しているロジック

`node_0X/src/applyPattern.cpp` の楽譜進行部（全ノード共通・差分は
`ProjectConfig.h` の `headRestBeats` のみ）:

```c
cyclePos = (firedBeatNo - 1) % CANON_CYCLE_BEATS;   // 56 拍のサイクル窓
local    = cyclePos - headRestBeats;                 // 0/8/16/24
if (0 <= local < kScoreLength) fire(kScore[local]);  // 曲長 32 拍
```

## 検証項目（すべて PASS）

1. 入り拍: node_02=拍1 / node_03=拍9 / node_04=拍17 / node_05=拍25（8 拍ずつズレ）
2. 各声部が周回内で楽譜 idx 0..31 を欠落なく連続消化
3. 輪唱性: 各声部は node_02 の完全な 8/16/24 拍遅れコピー
4. 終端: 拍 33〜56 で node_02 は次周回を始めず、node_05 が拍 56 で 1 周を
   終えた直後の拍 57 で node_02 が idx=0 から再入
5. 周期性: 2 周目（拍 57〜112）は 1 周目と同一パターン

## 3 台構成（node_05 不在）で試聴するときの注意

サイクル 56 拍は 4 台前提のため、3 台（node_02/03/04）では以下が**仕様**:

- 拍 33 で node_02 が沈黙（次周回を待つ）
- 拍 41〜48 は node_04 のみ
- 拍 49〜56 は**完全無音**（本来 node_05 だけが演奏する区間）
- 拍 57 で node_02 から 2 周目が始まる

この「尻すぼみ→8 拍無音」は輪唱が壊れたのではなく、最終声部の終端待ち。
