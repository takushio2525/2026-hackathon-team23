25G1053 齋藤翔太 / 担当箇所まとめ

楽譜データ形式・5パート振り分け・発音情報仕様

5台のArduinoをすべて楽器ノードとして扱い，指揮棒は別マイコンに分離する方針へ更新した担当範囲の要約です。

今回の変更点

旧方針では node_01 を指揮者ノード，node_02〜node_05 を楽器ノードとしていました。今回の話し合い後は，node_01〜node_05 の5台すべてを楽器ノードとし，指揮棒は別マイコンが担当します。
5台 Arduino楽器ノード
1台 指揮棒用の別マイコン
20B PCへ送るNOTE固定長

システム構成

指揮棒用マイコンはBPMとBEATをWiFi UDPで送信します。各Arduino楽器ノードは自分の楽譜を進め，発音情報をUSB SerialでPCへ送ります。

ユーザ指揮棒を振る
指揮棒用マイコンCTRL / BEATを送信
Arduino楽器ノードnode_01〜node_05
PC / ProcessingNOTEを受信して発音

ユーザ操作 → テンポ・拍の配信 → 楽譜進行 → 音色合成

齋藤担当範囲

WBS 担当内容 今回の扱い
116 楽譜データ形式の基本設計 5台分の楽譜をArduinoメモリ内のScoreEvent配列として持つ。
128 課題曲のパート振り分け 金管3声，低音1声，ドラム1声の5パートに分ける。
129 テンポ，音高，拍の表現 音高はMIDIノート番号，拍はdurationQ8，テンポは指揮棒用マイコンから届くBPMを使う。
130 ArduinoからPCへ送る音データ 各楽器ノードが20バイト固定長のNOTEをUSB Serialで送る。

5パート割り当て

node_01 金管1 / partId 0x01

0拍から主旋律を演奏する。

node_02 金管2 / partId 0x02

4拍遅れて主旋律を演奏する。

node_03 金管3 / partId 0x03

8拍遅れて主旋律を演奏する。

node_04 低音 / partId 0x04

拍の頭を支える低音伴奏を演奏する。

node_05 ドラム / partId 0x05

1拍ごとのリズム伴奏を演奏する。

楽譜データ形式

楽譜は各ArduinoのプログラムメモリにScoreEvent配列として埋め込みます。PC側から楽譜を読み込ませるのではなく，楽器ノード自身が自分のパートを進めます。

フィールド 意味
beatAtパート開始から何拍目に発火するか。
noteNumberMIDIノート番号。0の場合は休符。
velocity楽譜上の強さ。CTRLの値と合成できる。
durationQ8音の長さ。256を1拍として扱う。
flagsNoteOnや休符などの補助情報。

NOTE送信仕様

ArduinoからPCへ送るNOTEは20バイト固定長です。共通ヘッダ12バイトの後ろにNOTEペイロード8バイトを置きます。

offset field 意味
12partId0x01〜0x05で発音元パートを表す。
13noteNumberMIDIノート番号。休符は送信しない。
14velocity0〜127の強さ。
15gate1=NoteOn，0=NoteOff。
16durationMs発音予定長さ。
18予約0で埋める。

処理フロー

1. 起動WiFiとSerialを初期化し，CTRL受信を待つ。
2. 同期指揮棒用マイコンからBPMとBEATを受け取る。
3. 入り判定startBeatNoに達したパートから演奏する。
4. NOTE生成対象のScoreEventからNoteOn / NoteOffを作る。
5. 送信20バイトNOTEをPCへ送る。

BEAT欠損時は直前BPMから仮想BEATを作って自走し，実BEATが復帰したら通常演奏に戻します。
