%%  3. AIの出力の使い方
%% ============================================================
\begin{itembox}{3.\ AIの出力の使い方}
\cboxoff そのまま使った\quad
\cboxon 一部修正して使った\quad
\cboxoff 参考にしただけ

\vspace{6pt}
\textbf{選んだ理由：}
AIが挙げた原因と修正方針のうち、以下は妥当と判断して採用した：
\begin{itemize}
  \item 実測値 (0--1023) と振幅 (V単位) を同時に出力しているため、
        プロッタのY軸が整数側に引っ張られて振幅波形がつぶれる問題
  \item DCオフセットを固定値 (1.65V) で引いているため、
        実際のバイアスが違うと中心が0にならない問題 $\rightarrow$ EWMAによる動的追従に変更
  \item \texttt{delay} がないためシリアル送信バッファが溢れ、
        波形がガタつく問題 $\rightarrow$ \texttt{delayMicroseconds(500)} で 2\,kHz サンプリングに固定
  \item ボーレート 921600 は Arduino IDE のシリアルプロッタで
        取りこぼしが起きやすいので 115200 に下げる
\end{itemize}
一方で、以下は自分で調整・判断した：
\begin{itemize}
  \item EWMA の時定数 (\texttt{ALPHA\_DC}=0.001 / \texttt{ALPHA\_LP}=0.2) は
        AIの提案値をそのまま採用せず、マイク回路の特性を見ながら決める方針にした
  \item 出力ラベルは日本語だと環境依存で文字化けするリスクがあるので
        英字 (\texttt{signal}, \texttt{ref\_hi}, \texttt{ref\_lo}) に変更
  \item 参照電圧は基板依存（UNO なら 5.0V、XIAO/ESP32 なら 3.3V）なので、
        \texttt{\#define VREF} で切り替えやすくしコメントで明示
\end{itemize}
\end{itembox}

\begin{itembox}{4.\ 正しいと確認した方法}

想定している確認手順（実機テストで実施する）：
\begin{itemize}
  \item Arduino IDE でコンパイルが通ることを確認する
  \item シリアルモニタを 115200\,bps で開き、数値が流れることを確認する
  \item シリアルプロッタに切り替え、\texttt{signal} \texttt{ref\_hi} \texttt{ref\_lo} の3本が見えることを確認する
  \item 無音時に \texttt{signal} が 0 付近に収束することを観察する（DC追従の確認）
  \item マイクに声をかけると \texttt{signal} が上下に振動することを観察する
  \item コードを1行ずつ読み、EWMA 式 \(y_n = \alpha x + (1-\alpha) y_{n-1}\) の意味を説明できる状態にする
\end{itemize}
\end{itembox}

\textbf{検証予定ポイント（実機で確認する）：}
\begin{itemize}
  \item EWMA の初期値 \texttt{dc = 512.0f} が、
        起動直後に実際のDCバイアスに収束するまで何秒かかるか
  \item \texttt{ALPHA\_DC=0.001} で DC 追従が遅すぎて、
        長めの声など低周波成分まで消してしまわないか
  \item ボーレート 115200 + \texttt{delayMicroseconds(500)} + 3 トレース出力で
        シリアル送信が間に合うか（取りこぼしが起きないか）
\end{itemize}
\end{itembox}

%% ============================================================
%%  6. 自分で工夫した点
%% ============================================================
\begin{itembox}{6.\ 自分で工夫した点}
\begin{itemize}
  \item \texttt{HCK02\_01.ino} の冒頭に「例題2 (\texttt{work3.ino}) からの変更点」を
        コメントでまとめ、後から自分や他のメンバーが変更理由を追えるようにした
  \item シリアルプロッタ用ラベルを英字にし、Arduino IDE のバージョンや OS が
        変わっても文字化けしないようにした
  \item 参照電圧を \texttt{\#define VREF} で定数化し、UNO / XIAO 等 どの基板でも
        1行書き換えるだけで対応できる形にした
  \item EWMA の係数は定数名 (\texttt{ALPHA\_DC}, \texttt{ALPHA\_LP}) を
        目的別に分けて命名し、何を制御しているかが読んで分かるようにした
  \item \texttt{ref\_hi} / \texttt{ref\_lo} をあえて出力に混ぜることで、
        プロッタのY軸範囲が振幅レンジ（$\pm$1.65V など）に固定されるようにした
\end{itemize}
\end{itembox}
