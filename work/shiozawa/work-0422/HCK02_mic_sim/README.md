# HCK02_mic_sim — マイク入力擬似シリアル出力テストプログラム

## 何のフォルダか

`HCK02_02.pde` / `HCK02_03.pde` が想定する Arduino 側のマイク入力ストリームを、
**マイクが壊れている状態でも代替できるように Arduino 内部で合成して出力する**
テスト用スケッチ。

`HCK02_03.ino`（マイク値をそのまま `Serial.println()` するもの）を差し替えて使うと、
`.pde` 側を一切いじらずに「きれいに波形が取れた状態」をシミュレートできる。

## 内容

- `HCK02_mic_sim.ino` : メイン。`playSong()` と同じメロディ（C4, C4, G4, G4, A4, A4, G4）を
  サイン波 + 線形ダウン・エンベロープで合成し、0〜1023 の整数として 921600 baud で出力する。

## 仕様

| 項目 | 値 |
|------|----|
| Baud | 921600 |
| 出力フォーマット | `0`〜`1023` の整数を `\n` 区切り |
| 想定サンプリングレート | 5 kHz（`HCK02_03.pde` の `SAMPLE_RATE` と一致） |
| 1 ループ長 | 7.5 秒 |

内部時刻はサンプル番号 ÷ `SAMPLE_RATE` で進めているので、Serial 出力時間に
左右されず、FFT の周波数ビンに正しい位置でピークが立つ。

## 使い方

1. このスケッチを Arduino に書き込む（HCK02_03.ino と排他で OK）。
2. `HCK02_03.pde`（または `HCK02_02.pde`）の `PORT_NAME` を実機の usbmodem に合わせる。
3. `.pde` を実行する。マイクが繋がっていなくても波形と FFT が表示される。

`HCK02_03.pde` 側で `p` キーを押すと `playSong()` が走るが、こちらの Arduino 合成とは
独立に動く（同期しない）。スピーカーの音と画面波形がズレて見えるのは仕様。

## 不要になったら

マイクが直って実機波形が取れるようになったら、このフォルダごと削除してよい。
通常運用では `HCK02_03.ino` を Arduino に書き戻す。
