# 4. 要件分析

本章は第 2 回ミーティングで要求された **FBS / PBS / MOP** を、塩澤担当（Arduino 系
ファームウェア全般）の視点で整理する。チーム計画書に統合される前提で、楽器音色や発表準備
などチーム全体側の要素は含めない。

## 4.1 機能分解（FBS: Function Breakdown Structure）

Arduino 5 台で実現すべき機能を木構造で分解する。第 11〜12 章の詳細設計はこの分解を
1 対 1 で実装モジュールに対応させる。

```text
F. Arduino オーケストラ ファームウェア
├── F1. 指揮情報抽出（node_01）
│   ├── F1.1 センサ値取得（IMU 加速度 3 軸 + 角速度 3 軸）
│   ├── F1.2 信号前処理（ノイズ除去、重力成分分離）
│   ├── F1.3 拍検出（振り下ろしイベント検出）
│   ├── F1.4 テンポ推定（拍間隔 → BPM）
│   ├── F1.5 強弱推定（振幅 → velocity） ※ストレッチ
│   └── F1.6 演奏状態管理（Idle / Calibrating / Conducting / Fallback）
├── F2. 指揮情報配信（node_01）
│   ├── F2.1 制御コマンド送出（CTRL: BPM・velocity・演奏状態）
│   ├── F2.2 拍トリガ送出（BEAT: シーケンス番号・タイムスタンプ）
│   └── F2.3 ネットワーク接続維持（WiFi STA、再接続）
├── F3. 楽譜進行（node_02〜05）
│   ├── F3.1 指揮情報受信（CTRL / BEAT の UDP 受信）
│   ├── F3.2 楽譜データ保持（パート別・ROM 埋め込み）
│   ├── F3.3 BEAT 同期の音符進行（次ノートインデックスの更新）
│   ├── F3.4 発音イベント生成（NOTE: ノート番号・velocity・duration）
│   └── F3.5 パート固有ロジック（パート ID、開始オフセット＝輪唱の入りタイミング）
├── F4. 発音情報配信（node_02〜05 → PC）
│   ├── F4.1 NOTE パケット生成
│   └── F4.2 Processing への UDP 送信
└── F5. 共通基盤（全ノード）
    ├── F5.1 モジュール管理（IModule 登録・更新・3 フェーズループ）
    ├── F5.2 周期実行管理（ModuleTimer）
    ├── F5.3 ノード状態集約（SystemData）と設定集約（ProjectConfig）
    ├── F5.4 起動シーケンス・診断表示（LED）
    └── F5.5 エラーハンドリング・フォールバック
```

**ストレッチ項目**（F1.5）は必達機能の実装完了後に着手する。詳細設計では
インターフェースだけ用意し、中身はスタブで通す構成にする。

## 4.2 物理構成（PBS: Physical Breakdown Structure）

本設計書のスコープ（Arduino 系）に直接関わる物理要素を列挙する。
PC 側音響・スピーカはチーム全体側の PBS で扱うため、ここでは「インターフェース境界」のみ記す。

| 区分 | 要素 | 数量 | 備考 |
|---|---|---|---|
| P1.1 指揮者ノード | Arduino UNO R4 WiFi | 1 | 内蔵 IMU（LSM6DSOX 相当）を使用 |
| P1.1 | 内蔵 IMU | 1 | 加速度 ±4g / 角速度 ±2000 dps（デフォルト） |
| P1.1 | 内蔵 WiFi（ESP32-S3） | 1 | Arduino 標準 `WiFiS3` で制御 |
| P1.1 | 状態表示 LED | 1 | `LED_BUILTIN` を使用。外付け LED は運用時に検討 |
| P1.2 楽器ノード | Arduino UNO R4 WiFi | 4 | node_02〜05 は同一ハード構成 |
| P1.2 | 内蔵 WiFi | 4 | 受信専用（指揮者から） + 送信（PC へ NOTE） |
| P1.2 | 状態表示 LED | 4 | `LED_BUILTIN` |
| P2 ネットワーク | ローカル WiFi AP | 1 | スマホテザリング or 専用ルータ（運用で決定） |
| P2 | IP 割り当て | — | 共通層で吸収。静的 / DHCP は第 10 章で詳述 |
| P3 給電 | USB ケーブル（Type-C） | 5 | PC または USB ハブから給電 |
| P4 境界（参考） | PC（Processing 実行環境） | 1 | **本設計書スコープ外**。NOTE パケットを受信 |
| P4 境界（参考） | スピーカ | 1 | **本設計書スコープ外**。Processing から出力 |
| P4 境界（参考） | 指揮棒・装着具 | 1 | **本設計書スコープ外**。ハード運用時に決定 |

## 4.3 性能指標（MOP: Measure of Performance）

本ファームウェアの性能を測るための指標を、評価軸（[`docs/overview.md`](../../../../../docs/overview.md)
§評価軸、`references/lectures/HCK1_01_Kickoff_A.pdf` p.65）と対応させて定義する。
目標値は原案段階のたたき台で、実機計測後に V&V（第 13 章）で再調整する。

| ID | 指標 | 目標値 | 計測方法 | 対応する評価軸 |
|---|---|---|---|---|
| MOP-1 | **楽器間 同期誤差** | ≤ 30 ms | BEAT 到来時刻のログを 4 ノードで比較 | タイミング誤差 |
| MOP-2 | **指揮 → 楽器 通信遅延** | ≤ 10 ms | node_01 送出時刻と node_02 受信時刻の差 | 同期信号受信精度 |
| MOP-3 | **拍検出 正解率** | ≥ 90 %（定速指揮時） | 手動カウント vs 検出カウント | エンターテインメント性 |
| MOP-4 | **テンポ追従 遅延** | ≤ 2 拍 | BPM 階段変化を与えて追従時間を計測 | タイミング誤差 |
| MOP-5 | **パケロス耐性** | 5 % パケロスで演奏継続 | iperf / 擬似パケロスツールで検証 | 同期信号受信精度 |
| MOP-6 | **入力フェーズ CPU 時間** | ≤ 2 ms / 周期 | `micros()` 差分ログ | （内部指標） |
| MOP-7 | **起動時間** | ≤ 5 s（電源投入 → 演奏可能） | 電源投入〜WiFi 接続完了までを秒で計測 | （運用指標） |
| MOP-8 | **強弱制御 分解能** | 3 段階（p/mf/f）以上 | 指揮振幅を変えて velocity 出力を観察 | エンターテインメント性（ストレッチ） |
