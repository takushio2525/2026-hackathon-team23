\section*{1. 何に使ったか}
例題2 のプログラムをそのまま動かすと、シリアルプロッタで波形が見にくかった。
最大振幅・最小振幅といったラベル付きの値が一緒に出ていることや、表示の縦軸が
自動で動いてしまうことが原因だと考え、出力フォーマットの直し方について
生成AIに相談した。具体的には、シリアルプロッタで「波形が見やすい状態」に
するには何を出力すればよいか、という方針出しに使った。

\section*{2. AIへの指示（プロンプト）}
\begin{lstlisting}
例題2 のコードをシリアルプロッタで見ると、縦軸の範囲が勝手に変わって
波形が読めません。マイクの値を読み取って、波形が見やすく出るように
出力のしかたを直したいです。どういう出力にすればよいですか?
\end{lstlisting}

\section*{3. AIの出力の使い方}
\begin{itemize}
  \item \cboxoff そのまま使った
  \item \cboxon 一部修正して使った
  \item \cboxoff 参考にしただけ
\end{itemize}
$\rightarrow$ 上で選んだ理由：

AIからは「ラベル付きの文字列を消して、数値だけをカンマ区切りで出す」
「縦軸を固定するために、毎回 0 と 1023 のガイド線も一緒に出す」という
2つのアイデアが出てきた。これは自分の困っていたことに合っていたので採用した。
一方で、AI が出してきたサンプルコードには電圧変換やフィルタ処理など
今回必要のない部分が含まれていたので、その部分は削って、自分のコードでは
\texttt{analogRead} の生値・\texttt{0}・\texttt{1023} の3つだけを
カンマ区切りで出力する形に書き直した。

\section*{4. 正しいと確認した方法}
\begin{itemize}
  \item Arduino IDE でコンパイルが通ることを確認した
  \item シリアルモニタを 921600\,bps で開き、
        \texttt{<生値>,0,1023} の形式で1行ずつ流れていることを確認した
  \item シリアルプロッタに切り替え、上下に \texttt{0} と \texttt{1023} の
        ガイド線が引かれた状態で、中央の線がマイクに入る音に応じて動くことを
        観察した
  \item 例題2 と並べて表示し、波形が安定して読めるようになっていることを
        目で確認した
\end{itemize}

\section*{5. 問題や間違い}
\begin{itemize}
  \item \cboxon あった
  \item \cboxoff なかった
\end{itemize}

AI が最初に出してきたコードは、\texttt{Serial.print} で各値の前に
\texttt{"value:"} のような文字列ラベルを付ける形だった。これをそのまま使うと
1サンプルの送信バイト数が増えてしまい、結局シリアルプロッタの更新が
追いつかないという問題がありそうだった。実際に試してみて文字列ラベルは
要らないと判断し、整数だけをカンマで区切る形式に直した。

\section*{6. 自分で工夫した点}
\begin{itemize}
  \item 例題2 にあった電圧への換算 (\texttt{(float)d / (pow(2, RESOLUTION)-1) * 5.0})
        や中心化処理は今回の課題には不要だったので削除し、必要最小限の処理だけにした
  \item シリアルプロッタの縦軸が動くと波形が読めないので、毎ループで
        \texttt{0} と \texttt{1023} を一緒に送り、表示範囲が固定されるようにした
  \item サンプリングは \texttt{delay(1)} で約 1\,ms 周期にし、コードが読みやすい
        素直な書き方にした
\end{itemize}
