\section*{1. 何に使ったか}
授業（ハッカソン1 第2回）の例題2 で作成した「マイク入力を Serial Plotter に表示する」
プログラムを動かしたところ、最後のスライドにあったように波形が三角形のように
カクついて表示され、正弦波に見えない状態だった。提出課題1 はこの例題2 のプログラムを
「シリアルモニタ（プロッタ）に正しく波形が表示されるよう」修正する課題なので、
「何が原因で波形が崩れているのか」「どこを直せば波形らしく見えるのか」を生成AIに
相談した。

\section*{2. AIへの指示（プロンプト）}
\begin{lstlisting}
ハッカソン1 の例題2 のプログラム（Arduino でマイクの値を analogRead して
Serial.print("実測値:") などと一緒に出力し、Serial Plotter で見るもの）を
そのまま動かすと、波形がギザギザの三角形のようになって正弦波に見えません。
- 何が原因で波形が崩れているのか
- Serial Plotter に正しく波形が表示されるようにするには
  どこを直せばよいか
を教えてください。
\end{lstlisting}

\section*{3. AIの出力の使い方}
\begin{itemize}
  \item \cboxoff そのまま使った
  \item \cboxon 一部修正して使った
  \item \cboxoff 参考にしただけ
\end{itemize}
$\rightarrow$ 上で選んだ理由：

AI からは、原因として
(a) \texttt{delay(1)} ではサンプリング周期が約 1\,kHz しかなく、可聴音の波形を
表現するには粗すぎる、
(b) Serial Plotter は表示中の値の最大・最小に合わせて縦軸を勝手に動かすため、
1 系列だけ流すと振幅が変動するたびに見た目の波形が変形してしまう、
(c) \texttt{"実測値:"} のような文字ラベルや複数の値が並ぶと、Plotter が
意図しない系列を作って波形が読みにくくなる、という 3 点を提示された。
このうち (a) サンプリングを上げる、(b) 毎ループで縦軸固定用のガイドを一緒に
送る、(c) ラベルや余計な系列を消す、という方針はそのまま採用した。
一方で AI が最初に提示した「電圧 (\,V\,) に変換し中心化したうえで送る」サンプルは、
今回の目的（波形が見えること）には不要だったので使わず、生の ADC 値だけを送る形に
直した。

\section*{4. 正しいと確認した方法}
\begin{itemize}
  \item Arduino IDE で書き込み後、シリアルモニタを 921600\,bps で開き、
        \texttt{<生値>,0,1023} の形式で 1 行ずつ流れていることを確認した
  \item Serial Plotter に切り替えると、上下に \texttt{0} と \texttt{1023} の
        ガイド線が常に表示され、縦軸が動かなくなったことを確認した
  \item マイクに 440\,Hz の正弦波（スマホのチューナーアプリで再生）を聞かせ、
        ガタガタの三角形ではなく丸みのある波形として見えることを確認した
  \item 例題2 のときの \texttt{delay(1)} に戻して同じ音を入力すると
        波形が再びギザギザになることを確認し、サンプリング周期が
        原因であったことを再現実験で確かめた
\end{itemize}

\section*{5. 問題や間違い}
\begin{itemize}
  \item \cboxon あった
  \item \cboxoff なかった
\end{itemize}

AI が最初に出してきたコードは、各値の前に \texttt{"value:"} のような文字列ラベルを
\texttt{Serial.print} で付ける形だった。これでは上記 (c) の問題が残り、Plotter で
見ると複数系列扱いになって見にくくなったため、整数だけをカンマで区切る素朴な
形式に直した。また、サンプリング周期について「\texttt{delayMicroseconds(50)} なら
20\,kHz 取れる」と提案されたが、Arduino UNO R3 の \texttt{analogRead} 自体に約
100\,$\mu$s かかるため実際にはそこまで上がらない。手元で何度か試して、
\texttt{delayMicroseconds(100)} にしたとき波形が一番きれいに見えたので、その値に
落ち着けた。

\section*{6. 自分で工夫した点}
\begin{itemize}
  \item 例題2 にあった電圧換算（\texttt{(float) d / (pow(2,RESOLUTION)-1) * 5.0}）と
        中心化処理は今回不要だったので削除し、必要最小限の処理だけにした
  \item 毎ループで \texttt{0} と \texttt{1023} を一緒に送り、Serial Plotter の
        縦軸が動かないようにした
  \item サンプリングは \texttt{delayMicroseconds(100)} で素直な書き方にし、
        実測（マイクに 440\,Hz の音を入れる）で波形が滑らかに見えることを
        確認したうえで採用した
\end{itemize}
