\section*{1. 何に使ったか}
前回課題 (\texttt{HCK01\_03.pde}) の楽曲再生コードと、例題の Processing シリアル受信コード
(\texttt{work2\_processing.pde}) を組み合わせて、提出課題2（楽曲を再生しながらマイク波形を
Processing ウィンドウに描画）に仕上げる作業で生成AIを利用した。
\begin{itemize}
  \item Arduino 側 (\texttt{HCK02\_02.ino}) のシリアル出力フォーマットを、
        Processing が 1 行 1 サンプルで読めるように簡素化する方針の相談
  \item Processing 側 (\texttt{HCK02\_02.pde}) で楽曲バッファとマイク入力波形を
        同一ウィンドウに共存させるレイアウト案の洗い出し
  \item マイク波形を時間軸でスクロール表示するための循環バッファ実装のレビュー
\end{itemize}

\section*{3. AIの出力の使い方}
\begin{itemize}
  \item \cboxoff そのまま使った
  \item \cboxon 一部修正して使った
  \item \cboxoff 参考にしただけ
\end{itemize}
$\rightarrow$ 上で選んだ理由：

AIが挙げた設計のうち、以下は妥当と判断して採用した：
\begin{itemize}
  \item Arduino の出力を \texttt{Serial.println(d)} の 1 行 1 サンプル形式に絞り、
        Processing 側は \texttt{bufferUntil('\textbackslash n')} と
        \texttt{readStringUntil} で受信する形
  \item 上段に楽曲バッファ (L/R)、下段にマイク波形を置く 2 分割レイアウト
  \item マイク波形の循環バッファと、描画時に
        \texttt{(waveIdx + x) \% waveBuf.length} で参照する方式
\end{itemize}
一方で、AIから追加提案された「バイナリ 2 バイト + 同期マーカー形式での送信」は、
例題コードとの整合と実機でのデバッグしやすさを優先して採用を見送った。
Arduino の設定値（ボーレート 921600、サンプリング周期 150\,µs、ADC 10bit）は
\texttt{HCK02\_01.ino} で動作確認済みの値をそのまま踏襲した。

想定している確認手順：
\begin{itemize}
  \item Arduino IDE で \texttt{HCK02\_02.ino} がコンパイルでき、
        シリアルモニタ (921600\,bps) に数値が 1 行ずつ流れることを確認する
  \item Processing IDE で \texttt{HCK02\_02.pde} が起動し、
        下段に黄色いマイク波形が左 $\rightarrow$ 右に流れることを観察する
  \item キー \texttt{'p'} を押したときに、前回課題 (HCK01\_03) と同じ楽曲が再生され、
        上段に左右チャンネルの波形が見えることを確認する
  \item キー \texttt{'1'}〜\texttt{'6'} で音色が切り替わり、
        もう一度 \texttt{'p'} を押すと変わった音色で再生されることを確認する
  \item 楽曲をスピーカから流してマイクに入れたとき、下段のマイク波形が
        音に合わせて振幅するかを観察する
  \item コードを 1 行ずつ読んで、循環バッファのインデックス計算
        ($ (\texttt{waveIdx}+x) \bmod N $) の意味を自分の言葉で説明できる状態にする
\end{itemize}

\section*{6. 自分で工夫した点}
\begin{itemize}
  \item Arduino 側 (\texttt{HCK02\_02.ino}) は、動作確認済みの \texttt{HCK02\_01.ino} の
        サンプリング設定を踏襲しつつ、出力だけ Processing 向けの単一整数 + 改行に絞った
        （Y 軸ガイド線は Processing 側で描画するので不要）
  \item Processing 側 (\texttt{HCK02\_02.pde}) で、楽曲バッファ (minim の
        \texttt{out.left}/\texttt{out.right}) と、シリアル経由のマイク ADC 値を
        同じウィンドウに共存させ、上段：楽曲、下段：マイクで役割を明確にした
  \item マイク波形は循環バッファで保持し、描画時に \texttt{waveIdx} を起点に
        左 $\rightarrow$ 右へ古い順に描くことで、「右端が最新、左にスクロール」
        という自然な時系列表示を 1 パスで実装した
  \item ADC 値 (0--1023) を $[-1, +1]$ に正規化してから縦方向スケール
        \texttt{micAmp} を掛ける形にし、10bit 以外の解像度に切り替えても
        中心・振幅が壊れない構造にした
  \item \texttt{NumberFormatException} を握りつぶすことで、起動直後の半端な受信文字列
        （改行だけの行など）でスケッチが落ちないようにした
\end{itemize}
