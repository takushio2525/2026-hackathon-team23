# analyzer — 音源解析ツール + 編集スタジオ(Python バックエンド + ブラウザフロント)

楽器の **単音** 音源を 1 ファイル渡すと、Python (`librosa`) で徹底的に解析し、
[`../library_format.md`](../library_format.md) のインストゥルメント定義 JSON を出力する。
フロントはブラウザ。アップロード → 波形 / スペクトル / ADSR / 倍音 / ピッチの揺れを可視化 →
**鳴らしながら音色を編集**（ビブラート・倍音バランス・ノイズ/息・響き・ドライブ/コーラス/フィルタ/EQ …）→
調整後の JSON や、その音そのものを WAV で書き出し、までできる。原音との聴き比べ（A/B）付き。
金属楽器・フルート・オルガン・金属的な打楽器向けに、倍音・非調和性・アタック・EQ をまとめて整える
**単一楽器クオリティ向上モード**と、最大 12 音源を読み込んで音量バランスを調整し
WAV / JSON / `manifest.json` を ZIP でまとめたり、複数音を1つの音色パックJSONにまとめたりできる
**複数音源バランス調整モード**も備える。
フルートプロファイルは、原音本体サンプルを主音にし、息成分・タンギング・細めの倍音・控えめなビブラートを調整しやすくする。
録音頭のゆっくりしたフェードが強い音源でも、再合成時に笛の立ち上がりが鈍くなりすぎないよう ADSR を補正する。
オルガンプロファイルは、安定した持続音、短い立ち上がり、低ノイズ、倍音の厚みを調整しやすくする。
ドラム / 打楽器プロファイルでは、明確な基音が無い音でも落ちにくいよう、低域ピークを基準音として使い、
原音1打サンプルを主音として再生する。倍音・ノイズ解析は表示と微調整用に残し、キック / スネア /
ハイハット / クラッシュの再現はサンプル駆動で安定させる。

> 解析ロジック本体は [`analyzer.py`](analyzer.py)。サーバを介さず単体でも使える
> （`from analyzer import analyze_file` → `analyze_file("note.wav")` が dict を返す）。
> ブラウザ側は `static/index.html` + `static/engine.js`（Web Audio 再合成エンジン）+ `static/app.js`（UI）+ `static/studio.css`。

## セットアップ & 起動

### いちばん簡単（macOS）

`sound_lab/analyzer/start.command` を **Finder でダブルクリック**。初回だけ仮想環境の作成と依存
インストール（librosa を含むため数分）が走り、起動後はブラウザが自動で `http://127.0.0.1:5005`
を開く。止めるときはそのターミナルウィンドウで `Ctrl+C`。

> 「開発元を確認できません」と出たら、ファイルを右クリック →「開く」→「開く」で一度許可すればよい。

### 手動（macOS / Linux / Windows 共通）

```bash
cd sound_lab/analyzer
python3 -m venv .venv
source .venv/bin/activate           # Windows: .venv\Scripts\activate
pip install -r requirements.txt     # librosa を含むため初回は数分かかる
python app.py                       # 起動するとブラウザが自動で開く
# 開かなければ http://127.0.0.1:5005 を手で開く
```

`mp3` / `m4a` を読むには ffmpeg が必要（`brew install ffmpeg`）。`wav` / `flac` なら追加不要。
（原音 A/B 再生は、そのブラウザがデコードできる形式のときだけ使えます。`.wav` が無難）

> **「通信エラー: Failed to fetch」と出るとき** … サーバ（`python app.py` または `start.command`）が
> 起動していない、もしくは HTML ファイルを直接ダブルクリックして `file://` で開いている。必ず
> `http://127.0.0.1:5005` から開くこと。HTML 側にも同じ案内が出る。
> （解析せずスタジオだけ触りたいなら、画面の「内蔵デモ音色で試す」はサーバ無しでも動く。）

## 使い方

1. ブラウザで `http://127.0.0.1:5005`
2. 必要なら「楽器プロファイル」で `トランペット`、`フルート`、`オルガン`、`ドラム / 打楽器` を選ぶ。トランペット指定時は
   基音探索レンジと初期音色補正をトランペット向けに切り替え、原音先頭の短いアタック波形も保存する。
   さらに、定常部の短い原音ループ、わずかにピッチの違う倍音レイヤー、専用の管鳴り EQ で、
   金管らしいバズと管内反射の厚みを補う。
   フルート指定時は、原音本体を `tone_sample` として保存し、ブラウザスタジオではそれを主音として鳴らす。
   さらに、息やタンギングとして使う原音先頭と、持続音の芯として使う定常部ループも保存する。
   さらに、録音の表情として入った遅いフェードに引っ張られすぎないようアタックを短めに補正し、息ノイズを残す。
   オルガン指定時は、キークリック/立ち上がりとして使う原音先頭と、安定した持続部のループを保存する。
   ドラム指定時は基音検出に失敗しやすい音でも、低域ピークを便宜上の基準音にして解析し、原音1打を最大 3 秒保存する
3. 楽器の単音ファイル（`.wav` 推奨。`.flac/.ogg/.mp3/.aiff/.m4a` も可、〜32 MB）をドロップ → 自動で解析
4. 検出した基音・MIDI ノート・ADSR・倍音数・ビブラート/トレモロなどが表示され、波形・対数スペクトル
   （倍音マーカー付き）・ピッチの揺れ・倍音バーが描画される
5. **金属楽器向けクオリティ向上**を押すと、金属楽器らしい立ち上がり・明るい倍音・軽い非調和性・抜ける EQ が
   一括で入る。強すぎる場合は「明るさを少し戻す」や各スライダーで微調整する
6. **スタジオで鳴らしながら調整**:
   - 鍵盤クリック or PC キー（`z s x d c v g b h n j m` ／ 上段 `q 2 w 3 e r 5 t 6 y 7 u i`）を**押している間**鳴る
   - 「🔁 鳴らしっぱなし」で現在の音をドローン再生 → 手で各スライダーをいじる（変更は即反映）
   - 「🔊 原音を再生」で元の録音と聴き比べ（A/B）
   - 初期状態では、砂嵐のような残差ノイズや解析由来の細かい倍音揺れを足さず、ADSR で安定した音から始める
   - トランペット指定時は「原音アタックを重ねる」でタンギングを、「原音波形の芯を混ぜる」で原音に近い波形の芯を、
     「原音の伸びを重ねる」で定常部の管鳴りを、「トランペットの重なり」「トランペット管の共鳴」で唇のバズ感や管内反射の厚みを調整できる。
     強すぎる場合は下げる
   - ドラム指定時は「原音1打を主音にする」が初期状態で 100% になり、キック / スネア・タム /
     ハイハット / クラッシュ系の目安を表示する。「ドラムを太く整える」で打音向けの EQ と減衰に寄せられる
   - フルート指定時は「原音サンプルを主音にする」が初期状態で 100% になり、「フルート向けに強化」または
     「フルート向けに整える」で、原音本体、息成分、タンギング、細めの倍音、軽い空間感をまとめて設定できる。
     さらに目立たせたい場合は「息を強める」や「明るく抜ける」を使う
   - オルガン指定時は「オルガン向けに強化」または「オルガン向けに整える」で、短い立ち上がり、
     低ノイズ、安定したサステイン、倍音の厚み、控えめなコーラスをまとめて設定できる。
     さらに目立たせたい場合は「厚くする」や「教会風」を使う
   - パネル: 演奏/マスター ・ ピッチ&ビブラート ・ 倍音/音色（バーをドラッグして個別ゲインも編集）・
     トレモロ ・ ノイズ/息成分 ・ エンベロープ(ADSR) ・ 空間/響き(リバーブ) ・ エフェクト(ドライブ/コーラス/フィルタ) ・ ボディEQ
   - 各パネル右上の `↺` でその欄だけリセット。下部の「全パラメータを初期値に戻す」で全リセット
7. 仕上げたら **書き出し**:
   - 「⬇ 調整後のインストゥルメント JSON」→ `<名前>.tweaked.instrument.json`（Processing の `data/` に置けばその音色で鳴る。
     `brightness`・倍音の手動ゲイン・odd/even・倍音数・非調和性ミックス・ADSR は `harmonics[]`/`envelope` 本体に畳み込まれる。
     リバーブ等の演出は `fx` 欄に入り、Processing 現行版は無視する）
   - 「⬇ いまの音を WAV で書き出し」→ 鍵盤で選んだ音程・指定秒数を**全 FX 込み**でレンダリングして `.wav` に
   - 「保存した JSON を読み込む」で、前に書き出した定義を読み戻して編集を続けられる

### 複数音源バランス調整モード

画面上部の「複数音源バランス調整」を選び、最大 12 個の単音ファイルをまとめて選択する。
「複数音源を解析」を押すと各音源を順番に解析し、録音ピークを目安に音量を自動でそろえる。
必要に応じて各カードの音量スライダーを動かし、「複数音源を金属楽器向けに強化」で全トラックに同じ金属楽器向け補正をかける。
ファイル名に `flute_C4.wav` や `organ_C3.wav` のような音名が入っている場合は、その音名を優先して音色パックの `note_map` に使う。

フルートやオルガンを複数音で作る場合は、次のように命名してまとめて選択するとよい。

- `flute_C4.wav`
- `flute_D4.wav`
- `flute_E4.wav`
- `organ_C3.wav`
- `organ_E3.wav`
- `organ_G3.wav`

「1つの音色パックJSON」は、解析済みの各音源を次の形式で1ファイルにまとめる。

- `format`: `sound_lab.multinote_instrument/1`
- `note_map`: `60=C4`, `62=D4` のような MIDI 番号と元ファイル名の対応
- `sounds`: 各 MIDI 番号ごとの調整済みインストゥルメント定義

「代表音色JSON」は、複数音源の倍音・エンベロープ・ノイズ・1周期波形を平均し、通常の
`sound_lab.instrument/1` として 1 ファイルに書き出す。既存の単音インストゥルメント読み込みで扱いやすい一方、
音程ごとの違いを使い分けるわけではないため、C4〜B4 の各音をより自然に鳴らしたい場合は
「1つの音色パックJSON」のほうが有利。

「ZIP で書き出し」は、各音源について以下を 1 つの ZIP にまとめる。

- `<番号>_<名前>_<音名>.wav`: バランス調整後の WAV
- `<番号>_<名前>.tweaked.instrument.json`: 同じ補正を畳み込んだインストゥルメント定義
- `manifest.json`: 元ファイル名、音名、音量倍率、出力ファイル名の一覧

## 解析でやっていること（`analyzer.py`）

| 項目 | 手法 |
|---|---|
| 無音トリム（前処理） | フレーム RMS から、先頭はピークの `-20dB` 未満、末尾は `-50dB` 未満を無音とみなしてカット（立ち上がり手前 20ms・末尾 80ms は余白を残す）。デジタル無音だけでなく、録音のルームトーンや準備音・息も切る。削った秒数は `features.trimmed_lead_sec` / `trimmed_trail_sec` に出る |
| 基音検出 | `librosa.pyin`（有声フレームの中央値）。失敗時は振幅最大付近の自己相関にフォールバック。f0 のフレームごとの推移も保持（ビブラート解析・ピッチ揺れ描画に使う） |
| ドラム基準音 | `ドラム / 打楽器` プロファイル時は pyin を使わず、アタック直後のスペクトルから低域寄りに重み付けしたピークを拾う。これは音名表示と移調のための便宜的な基準で、厳密な基音ではない |
| 振幅エンベロープ | フレーム RMS（200 Hz 解像度）をピーク 1.0 で正規化 |
| ADSR + ループ点 | アタックは「ピークの 85% に最初に到達するまで」（=本体に強弱の揺らぎがあっても立ち上がりだけ拾える）。以降は そのアタック終了点を起点に、中盤レベル＝サステイン、`-26dB` 交差点までを音の有効区間として D / R / ループ区間を当てはめ |
| 倍音列 | 定常部の長尺ゼロ詰め FFT で `n·f0` 近傍（±3.5%）のピークを抽出（放物線補間で周波数微修正）。振幅・位相・実周波数比 |
| 倍音ごとの時間変化 | STFT（n\_fft=8192）で各倍音ビンのマグニチュード推移を 32 点に圧縮 |
| 非調和性 B | 検出倍音の実周波数を `f_n ≈ n·f0·√(1+B·n²)` に振幅重み付き最小二乗フィット |
| ビブラート / トレモロ | f0 トラックを「中央値からのセント差」にして中央 20〜80% を FFT → 3〜9 Hz 帯の卓越成分があれば ビブラート（速さ・全幅セント・始まりの遅れ）。振幅エンベロープでも同様にして トレモロ。`modulation` に格納（弱ければ `detected:false`・0） |
| 残差ノイズ | STFT 上で倍音まわりのビンをマスク → 残差の RMS から レベル・時間包絡、平均スペクトルを 9 帯域に集約して色（息/弓/打撃ノイズなど） |
| 単一周期波形 | 定常部の立ち上がりゼロクロスから 1 周期を切り出し 1024 点にリサンプル（ウェーブテーブル用） |
| 原音アタック | トランペット指定時はトリム後の原音先頭 0.18 秒、フルート指定時は 0.16 秒、オルガン指定時は 0.08 秒、ドラム指定時は 0.45 秒を 22.05 kHz に落として保存。ブラウザスタジオで通常の再合成に短く重ね、白色ノイズを増やさずタンギング、息の出だし、キークリック、打面のクリック、胴鳴りの出だしを補う |
| 持続音定常ループ | トランペット / フルート / オルガン指定時は定常部から約 0.22 秒を基音周期の整数倍に近い長さで切り出し、44.1 kHz で `sustain_sample` に保存する。ブラウザスタジオでは薄くループして、倍音加算だけでは出にくい唇のバズ、管鳴り、息の芯、オルガンの定常倍音を補う |
| 持続楽器原音本体 | フルート指定時はトリム後の原音本体を最大 4 秒、22.05 kHz で `tone_sample` に保存する。ブラウザスタジオではこれを主音として鳴らし、倍音/ノイズ合成は薄い補助にする |
| ドラム原音1打 | ドラム指定時はトリム後の原音本体を最大 3 秒、44.1 kHz で `drum_sample` に保存する。ブラウザスタジオではこれを主音として鳴らし、倍音/ノイズ合成は薄い補助にする。ハイハットの細かい高域とキックのアタックを残すため、本体サンプルはダウンサンプルしない |
| ドラム特徴量 | ドラム指定時は低域 / 中域 / 高域のエネルギー比、減衰時間、トランジェント強度から、キック系・スネア/タム系・ハイハット系・クラッシュシンバル系の目安を `features` に保存する |
| 特徴量（表示用） | スペクトル重心・ロールオフ・帯域幅・ゼロ交差率・スペクトル平坦度 |

`modulation` の検出はやや厳しめ（はっきり周期的でないと拾わない）。スタジオ側では、解析元より不自然に音が揺れないように、
検出値は参考表示にとどめ、初期再生ではビブラート/トレモロの深さを 0 にして安定した音から始める。
必要なときだけ「検出値に合わせる」や深さスライダーで揺れを付けられる。
同じ理由で、残差ノイズ量と倍音ごとの時間変化も初期状態では 0 / オフにしてある。
ノイズの質感や倍音の時間変化を再現したい場合だけ、該当スライダーやチェックを上げる。

## 入力のコツ

- **必ず 1 音だけ**。和音・フレーズ・残響まみれの録音は基音検出が破綻する
- 前後の無音は自動でカットされる（画面に「元 X s → 先頭 Y s カット」と表示）。気になるならカットの強さは
  `analyzer.py` の `TRIM_LEAD_DB`（既定 20）/ `TRIM_TRAIL_DB`（既定 50）で調整できる。`TRIM_LEAD_DB` を大きく
  すると控えめに、小さくすると積極的にカットする
- サンプルレートは内部で 44.1 kHz に統一されるので元は何でもよい
- うまく基音が取れないときは、`analyzer.py` の `FMIN` / `FMAX` を楽器の音域に合わせて狭める
- ドラム / 打楽器では単音というより「1 打だけ」を渡す。ループやフィルインを渡すと、複数打音が混ざってアタック保存や
  音量包絡が不自然になる

## ディレクトリ

|---|---|
