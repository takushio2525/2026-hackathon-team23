# analyzer — 音源解析ツール + 編集スタジオ(Python バックエンド + ブラウザフロント)

楽器の **単音** 音源を 1 ファイル渡すと、Python (`librosa`) で徹底的に解析し、
[`../library_format.md`](../library_format.md) のインストゥルメント定義 JSON を出力する。
フロントはブラウザ。アップロード → 波形 / スペクトル / ADSR / 倍音 / ピッチの揺れを可視化 →
**鳴らしながら音色を編集**（ビブラート・倍音バランス・ノイズ/息・響き・ドライブ/コーラス/フィルタ/EQ …）→
調整後の JSON や、その音そのものを WAV で書き出し、までできる。原音との聴き比べ（A/B）付き。

> 解析ロジック本体は [`analyzer.py`](analyzer.py)。サーバを介さず単体でも使える
> （`from analyzer import analyze_file` → `analyze_file("note.wav")` が dict を返す）。
> ブラウザ側は `static/index.html` + `static/engine.js`（Web Audio 再合成エンジン）+ `static/app.js`（UI）+ `static/studio.css`。

## セットアップ & 起動

### いちばん簡単（macOS）

`sound_lab/analyzer/start.command` を **Finder でダブルクリック**。初回だけ仮想環境の作成と依存
インストール（librosa を含むため数分）が走り、起動後はブラウザが自動で `http://127.0.0.1:5005`
を開く。止めるときはそのターミナルウィンドウで `Ctrl+C`。

> 「開発元を確認できません」と出たら、ファイルを右クリック →「開く」→「開く」で一度許可すればよい。

### 手動（macOS / Linux / Windows 共通）

```bash
cd sound_lab/analyzer
python3 -m venv .venv
source .venv/bin/activate           # Windows: .venv\Scripts\activate
pip install -r requirements.txt     # librosa を含むため初回は数分かかる
python app.py                       # 起動するとブラウザが自動で開く
# 開かなければ http://127.0.0.1:5005 を手で開く
```

`mp3` / `m4a` を読むには ffmpeg が必要（`brew install ffmpeg`）。`wav` / `flac` なら追加不要。
（原音 A/B 再生は、そのブラウザがデコードできる形式のときだけ使えます。`.wav` が無難）

> **「通信エラー: Failed to fetch」と出るとき** … サーバ（`python app.py` または `start.command`）が
> 起動していない、もしくは HTML ファイルを直接ダブルクリックして `file://` で開いている。必ず
> `http://127.0.0.1:5005` から開くこと。HTML 側にも同じ案内が出る。
> （解析せずスタジオだけ触りたいなら、画面の「内蔵デモ音色で試す」はサーバ無しでも動く。）

## 使い方

1. ブラウザで `http://127.0.0.1:5005`
2. 楽器の単音ファイル（`.wav` 推奨。`.flac/.ogg/.mp3/.aiff/.m4a` も可、〜32 MB）をドロップ → 自動で解析
3. 検出した基音・MIDI ノート・ADSR・倍音数・ビブラート/トレモロなどが表示され、波形・対数スペクトル
   （倍音マーカー付き）・ピッチの揺れ・倍音バーが描画される
4. **スタジオで鳴らしながら調整**:
   - 鍵盤クリック or PC キー（`z s x d c v g b h n j m` ／ 上段 `q 2 w 3 e r 5 t 6 y 7 u i`）を**押している間**鳴る
   - 「🔁 鳴らしっぱなし」で現在の音をドローン再生 → 手で各スライダーをいじる（変更は即反映）
   - 「🔊 原音を再生」で元の録音と聴き比べ（A/B）
   - パネル: 演奏/マスター ・ ピッチ&ビブラート ・ 倍音/音色（バーをドラッグして個別ゲインも編集）・
     トレモロ ・ ノイズ/息成分 ・ エンベロープ(ADSR) ・ 空間/響き(リバーブ) ・ エフェクト(ドライブ/コーラス/フィルタ) ・ ボディEQ
   - 各パネル右上の `↺` でその欄だけリセット。下部の「全パラメータを初期値に戻す」で全リセット
5. 仕上げたら **書き出し**:
   - 「⬇ 調整後のインストゥルメント JSON」→ `<名前>.tweaked.instrument.json`（Processing の `data/` に置けばその音色で鳴る。
     `brightness`・倍音の手動ゲイン・odd/even・倍音数・非調和性ミックス・ADSR は `harmonics[]`/`envelope` 本体に畳み込まれる。
     リバーブ等の演出は `fx` 欄に入り、Processing 現行版は無視する）
   - 「⬇ いまの音を WAV で書き出し」→ 鍵盤で選んだ音程・指定秒数を**全 FX 込み**でレンダリングして `.wav` に
   - 「保存した JSON を読み込む」で、前に書き出した定義を読み戻して編集を続けられる

## 解析でやっていること（`analyzer.py`）

| 項目 | 手法 |
|---|---|
| 無音トリム（前処理） | フレーム RMS から、先頭はピークの `-20dB` 未満、末尾は `-50dB` 未満を無音とみなしてカット（立ち上がり手前 20ms・末尾 80ms は余白を残す）。デジタル無音だけでなく、録音のルームトーンや準備音・息も切る。削った秒数は `features.trimmed_lead_sec` / `trimmed_trail_sec` に出る |
| 基音検出 | `librosa.pyin`（有声フレームの中央値）。失敗時は振幅最大付近の自己相関にフォールバック。f0 のフレームごとの推移も保持（ビブラート解析・ピッチ揺れ描画に使う） |
| 振幅エンベロープ | フレーム RMS（200 Hz 解像度）をピーク 1.0 で正規化 |
| ADSR + ループ点 | アタックは「ピークの 85% に最初に到達するまで」（=本体に強弱の揺らぎがあっても立ち上がりだけ拾える）。以降は そのアタック終了点を起点に、中盤レベル＝サステイン、`-26dB` 交差点までを音の有効区間として D / R / ループ区間を当てはめ |
| 倍音列 | 定常部の長尺ゼロ詰め FFT で `n·f0` 近傍（±3.5%）のピークを抽出（放物線補間で周波数微修正）。振幅・位相・実周波数比 |
| 倍音ごとの時間変化 | STFT（n\_fft=8192）で各倍音ビンのマグニチュード推移を 32 点に圧縮 |
| 非調和性 B | 検出倍音の実周波数を `f_n ≈ n·f0·√(1+B·n²)` に振幅重み付き最小二乗フィット |
| ビブラート / トレモロ | f0 トラックを「中央値からのセント差」にして中央 20〜80% を FFT → 3〜9 Hz 帯の卓越成分があれば ビブラート（速さ・全幅セント・始まりの遅れ）。振幅エンベロープでも同様にして トレモロ。`modulation` に格納（弱ければ `detected:false`・0） |
| 残差ノイズ | STFT 上で倍音まわりのビンをマスク → 残差の RMS から レベル・時間包絡、平均スペクトルを 9 帯域に集約して色（息/弓/打撃ノイズなど） |
| 単一周期波形 | 定常部の立ち上がりゼロクロスから 1 周期を切り出し 1024 点にリサンプル（ウェーブテーブル用） |
| 特徴量（表示用） | スペクトル重心・ロールオフ・帯域幅・ゼロ交差率・スペクトル平坦度 |

`modulation` の検出はやや厳しめ（はっきり周期的でないと拾わない）。スタジオ側では検出値がスライダーに初期値として
入り、深さを 0 にすれば「ビブラート/トレモロを消す」、上げれば「強める」、フラットな音でも 0 から上げれば「付ける」ができる。

## 入力のコツ

- **必ず 1 音だけ**。和音・フレーズ・残響まみれの録音は基音検出が破綻する
- 前後の無音は自動でカットされる（画面に「元 X s → 先頭 Y s カット」と表示）。気になるならカットの強さは
  `analyzer.py` の `TRIM_LEAD_DB`（既定 20）/ `TRIM_TRAIL_DB`（既定 50）で調整できる。`TRIM_LEAD_DB` を大きく
  すると控えめに、小さくすると積極的にカットする
- サンプルレートは内部で 44.1 kHz に統一されるので元は何でもよい
- うまく基音が取れないときは、`analyzer.py` の `FMIN` / `FMAX` を楽器の音域に合わせて狭める

## ディレクトリ

| パス | 内容 |
|---|---|
| `start.command` | macOS 用ワンクリック起動（venv 自動作成 → サーバ起動 → ブラウザを開く） |
| `app.py` | Flask サーバ（`/`, `/static/<f>`, `/analyze`, `/samples/<f>`。起動時にブラウザを自動で開く） |
| `analyzer.py` | 解析ロジック本体（サーバ非依存） |
| `static/index.html` | フロントのページ枠 + 重要スタイルのみ |
| `static/engine.js` | リアルタイム再合成 + エフェクトのエンジン（`SL.LiveSynth`。WAV 書き出しもここ） |
| `static/app.js` | アップロード・解析・可視化・編集パネル・鍵盤などの UI |
| `static/studio.css` | スタイル（Glass Pastel） |
| `samples/` | 動作確認用の音源置き場（中身は任意。`/samples/<ファイル名>` で配信） |
| `requirements.txt` | Python 依存 |

不要になったら `sound_lab/` ごと削除して構わない。
