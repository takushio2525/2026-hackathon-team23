# references — 外部から与えられた参考資料の置き場

プロジェクトで参照する**外部資料**をまとめるディレクトリ。

`docs/` が「チームが**自分たちで書く**設計ドキュメント」であるのに対し、
`references/` は「**外から与えられた資料**」を集めておく場所。

## こんなときに使う

- 授業で配布された資料（PDF・スライド等） → [`lectures/`](lectures/)
- 部品のデータシート・マニュアル
- 参考にした論文・外部仕様書・プロトコル仕様

## 構成

| パス | 内容 |
|---|---|
| [`lectures/`](lectures/) | 授業で配布された資料 |

必要に応じて `datasheets/`, `papers/` などサブフォルダを増やしてよい。

## 著作権上の注意（重要）

授業資料・データシート・論文などは、**配布元の著作物**である場合が多い。
以下のルールをチームで必ず確認すること：

- **Public リポジトリには資料本体を push しない**（再配布にあたる恐れがある）
- Public 運用なら `.gitignore` に `references/lectures/` などを追加し、
  資料本体はチーム内の別手段（Google Drive・Slack 等）で共有する
- Private リポジトリで、かつチーム内利用に限定する場合のみコミットしてよい

判断に迷ったら、配布元（教員・メーカー等）の利用規約を確認すること。

## 不要な班は

このディレクトリごと削除してよい。
