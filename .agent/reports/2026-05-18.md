## 3. 改善・新機能アイデア

### 3.1 共通スクリプトで「3 台コピー」を自動化（S, Med）
**なぜ必要か**: 2.1 の sed 過誤・2.2 の重複保守が示すように、3 ノード分の `score_data.cpp` を

### 3.2 `latexmk` 自動チェック用 GitHub Actions（M, Low-Med）
**なぜ必要か**: 1.6 の「PDF コミット忘れ」を機械的に検出。CI で Docker
（`ghcr.io/paperist/texlive-ja:debian`）を呼んで `latexmk` を走らせ、生成 PDF と
コミット内 PDF を `cmp` で比較するワークフロー。LuaLaTeX フォント依存ものは
`% !TEX program = lualatex` 検出して skip。**工数 M, 優先度 Low-Med**。

### 3.3 `partId` 範囲を `OrcProtocol.h` の列挙体で明示（S, Low）

パース部もスイッチで網羅性をチェックできる。**工数 S, 優先度 Low**。

### 3.4 夜間レビューレポートの「前提コミット sha」を自動埋め込み（XS, Med）
