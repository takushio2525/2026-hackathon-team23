# プロジェクト概要 — Arduino オーケストラ（チーム23）

## テーマ

**Arduino UNO R4 WiFi × 5台による楽器間同期演奏システム**。
指揮者マイコンが IMU ジェスチャを解釈し、4台の楽器マイコンへ演奏制御コマンドを
UDP で配信する。楽器マイコンは音符情報を Processing（PC）に送り、
PC 側で金管楽器の音色合成・再生を行う。

## ゴール

### 必達

1. 指揮者 1 台 + 楽器 4 台の **5 台同期演奏** が成立する
2. 輪唱曲（主旋律 3 + リズム 1 以上）を演奏できる
3. **可変テンポ**に対応する（指揮の速度に追従）

### ストレッチ

4. 指揮ジェスチャで **強弱（ベロシティ）** を制御できる
5. 指揮ジェスチャで **ビブラート** を制御できる
6. 見た目のエンタメ要素（LED・装飾）を付加

## 独創性のポイント

- 指揮者マイコン内蔵 **IMU** による**ジェスチャ同期**
  - 振る速度 → テンポ
  - 振幅 → 強弱
  - 細かい振動 → ビブラート
- 詳細: [`decisions/0003-conductor-imu.md`](decisions/0003-conductor-imu.md)

## 評価軸（授業資料より）

| 評価項目 | 内容 | 本チームの狙い |
|---|---|---|
| 同期信号の受信精度 | 正しく受信できているか | UDP＋独自プロトコルで欠損時も復帰 |
| タイミング誤差 | 演奏ズレの短さ | 楽器間 ≤30 ms を目安 |
| エンターテインメント性 | 見た目の楽しさ・華やかさ | 指揮ジェスチャで「指揮者感」を演出 |
| 楽器音らしさ | 目的楽器に聞こえるか | Processing で金管の倍音構造＋ADSR |

詳細根拠: [`../references/lectures/HCK1_01_Kickoff_A.pdf`](../references/lectures/HCK1_01_Kickoff_A.pdf)（p.65）

## 制約条件

- ハードウェア: Arduino UNO R4 WiFi **5 台**（人数分 − 1）
- 各楽器は **異なるパート**を演奏する（同一パート複数台は NG）
- 実験室の機材は貸し出し可。はんだ付けは各自で実施
- 無い機材はチームで相談のうえ購入
