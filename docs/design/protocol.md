# 通信方針（概略）

現時点では通信の方向性だけ決めている。具体的なパケット構造・ポート番号・
同期手順は実装フェーズで設計する。決定経緯は
[`../decisions/0002-udp-original-protocol.md`](../decisions/0002-udp-original-protocol.md) 参照。

## 決まっていること

- ベースは **UDP**（Arduino UNO R4 WiFi の標準ライブラリで扱える範囲）
- 流れは以下の2系統：
  - **指揮者 → 楽器ノード**：1対多で演奏制御情報を流す
  - **楽器ノード → PC**：各楽器が発音情報を PC に流す

## 未定（実装フェーズで詰める）

- パケットの具体的な構造（ヘッダ・フィールド・サイズ）
- メッセージ種別の分け方（制御／拍／発音／開始停止 など）
- ポート番号・IP アドレスの割り当て
- 同期の取り方（マスタークロック・拍補正・内挿の有無）
- パケロス・ジッタへの対処方針
- プロトコルのバージョン管理方法
