# 通信プロトコル仕様（UDP 独自プロトコル）

本プロジェクトの楽器間通信プロトコル仕様。
通信方式の決定経緯は [`../decisions/0002-udp-original-protocol.md`](../decisions/0002-udp-original-protocol.md) 参照。

## 前提

- **物理層**: WiFi 802.11 (2.4GHz)
- **トランスポート**: UDP
- **トポロジ**: 指揮者→楽器 はブロードキャスト（or マルチキャスト）、
  楽器→PC はユニキャスト
- **エンディアン**: リトルエンディアン（Arduino のネイティブ）
- **ポート**:
  - CTRL（指揮→楽器）: **UDP 5005**
  - NOTE（楽器→PC）: **UDP 5006**

## アドレス設計

| ノード | IP（例） | 役割 |
|---|---|---|
| 指揮者 | 192.168.10.10 | CTRL 送信元 |
| 楽器1（パートA） | 192.168.10.11 | CTRL 受信、NOTE 送信 |
| 楽器2（パートB） | 192.168.10.12 | 同上 |
| 楽器3（パートC） | 192.168.10.13 | 同上 |
| 楽器4（パートD） | 192.168.10.14 | 同上 |
| PC（Processing） | 192.168.10.20 | NOTE 受信 |
| ブロードキャスト | 192.168.10.255 | CTRL 配信先 |

※ 実機セットアップ時に確定する。スイッチング HUB／AP 含めた構成は実装フェーズで詰める。

## メッセージ定義

### 共通ヘッダ（4 byte）

| フィールド | 型 | サイズ | 説明 |
|---|---|---|---|
| `magic` | uint8 | 1 | 識別子 `0xAC`（"AC" = Arduino Conductor） |
| `version` | uint8 | 1 | プロトコルバージョン（現行 `0x01`） |
| `msg_type` | uint8 | 1 | メッセージ種別（下表） |
| `seq` | uint8 | 1 | シーケンス番号（0〜255、ロールオーバ可） |

### メッセージ種別

| `msg_type` | 名称 | 方向 | 用途 |
|---|---|---|---|
| `0x01` | CTRL | 指揮 → 楽器（BC） | テンポ・強弱・ビブラート連続値 |
| `0x02` | BEAT | 指揮 → 楽器（BC） | 拍頭タイミング通知（同期用） |
| `0x10` | NOTE_ON | 楽器 → PC | 音符発音 |
| `0x11` | NOTE_OFF | 楽器 → PC | 音符消音 |
| `0x20` | HELLO | 楽器 → 指揮 | 起動通知（任意） |
| `0xF0` | STOP | 指揮 → 楽器（BC） | 演奏停止 |
| `0xF1` | START | 指揮 → 楽器（BC） | 演奏開始 |

### CTRL パケット（指揮 → 楽器）

ヘッダ 4 byte + ペイロード 8 byte = **12 byte**

| Offset | フィールド | 型 | 単位 / 値域 | 意味 |
|---|---|---|---|---|
| 4 | `tempo_bpm_x10` | uint16 | 400〜2400（40.0〜240.0 BPM） | テンポ × 10 |
| 6 | `dynamics` | uint8 | 0〜127 | 強弱（MIDI ベロシティ準拠） |
| 7 | `vibrato_depth` | uint8 | 0〜127 | ビブラート深さ |
| 8 | `vibrato_rate_x10` | uint8 | 0〜200（0.0〜20.0 Hz） | ビブラート速さ × 10 |
| 9 | `expression` | uint8 | 0〜127 | 表現量（将来拡張、未使用なら 0） |
| 10 | `flags` | uint8 | bit field | 下表 |
| 11 | `reserved` | uint8 | 0 | 予約（パディング） |

#### `flags` ビット定義

| Bit | 名称 | 意味 |
|---|---|---|
| 0 | `MUTE_ALL` | 全楽器ミュート |
| 1 | `FERMATA` | フェルマータ（拍進行を一時停止） |
| 2-7 | 予約 | 0 |

### BEAT パケット（指揮 → 楽器）

ヘッダ 4 byte + ペイロード 4 byte = **8 byte**

| Offset | フィールド | 型 | 意味 |
|---|---|---|---|
| 4 | `beat_index` | uint16 | 曲頭からの拍番号（小節番号 × 拍子 + 拍位置） |
| 6 | `subdivision` | uint8 | 1拍内の細分位置（0=拍頭、ストレッチ用途） |
| 7 | `reserved` | uint8 | 0 |

### NOTE_ON / NOTE_OFF パケット（楽器 → PC）

ヘッダ 4 byte + ペイロード 6 byte = **10 byte**

| Offset | フィールド | 型 | 意味 |
|---|---|---|---|
| 4 | `part_id` | uint8 | パート識別子（0=A, 1=B, 2=C, 3=D） |
| 5 | `pitch_midi` | uint8 | MIDI ノート番号（21〜108） |
| 6 | `velocity` | uint8 | ベロシティ（0〜127、NOTE_OFF時は 0） |
| 7 | `vibrato_depth` | uint8 | ビブラート深さ（合成パラメータ） |
| 8 | `vibrato_rate_x10` | uint8 | ビブラート速さ × 10 |
| 9 | `flags` | uint8 | 予約 |

## 同期方式

### 拍同期（BEAT パケット駆動）

1. 指揮者は内部に**論理時刻（拍カウンタ）**を持つ
2. 指揮 IMU から得たテンポに従って拍カウンタを進める
3. 拍頭ごとに BEAT パケットをブロードキャスト
4. 楽器側はローカルの拍カウンタを BEAT パケットで上書き補正
5. BEAT パケットが落ちた場合、楽器側は最後の拍時刻＋ローカル時計で内挿

### CTRL パケット送出周期

- 通常: **20 ms 周期**（50 Hz）で連続値を流し続ける
- ジェスチャ変化時: 即時送出（変化検出ベース）

### NOTE パケット送出タイミング

- 楽器側で拍カウンタが楽譜の発音タイミングに到達した瞬間に NOTE_ON
- 音符長経過後に NOTE_OFF
- レイテンシ目標: 拍タイミングに対して ±10 ms 以内

## エラー処理

| エラー | 検出方法 | 対処 |
|---|---|---|
| パケロス（CTRL） | seq の不連続 | 直前の値を保持して演奏継続 |
| パケロス（BEAT） | beat_index の飛び | 飛び先まで内挿で進める |
| 不正パケット | magic / version 不一致 | 破棄してログ出力 |
| 指揮者ロスト | CTRL/BEAT が 500 ms 以上途絶 | デフォルトテンポでフリーラン |

## タイミング要件

| 項目 | 目標値 |
|---|---|
| 指揮 → 楽器 配信遅延 | ≤ 10 ms |
| 楽器 → PC 発音遅延 | ≤ 20 ms |
| 楽器間の発音ジッタ | ≤ 30 ms（人間が「ズレ」を感知する目安） |
| BEAT 送出周期 | テンポ依存（120BPM なら 500 ms） |
| CTRL 送出周期 | 20 ms（50 Hz） |

## 実装メモ（参考）

- Arduino: `WiFiUDP` ライブラリ標準
- Processing: `oscP5` ではなく `hypermedia.net.UDP` 等で生 UDP 受信
- パケットサイズはどれも 16 byte 以下に収まるため MTU 問題なし
