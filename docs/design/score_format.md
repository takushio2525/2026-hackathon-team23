# 楽譜データフォーマット

各楽器マイコンが内部に持つ**楽譜データ**の形式。輪唱曲（4声）に対応する。

## 設計方針

- 楽器マイコン（Arduino UNO R4 WiFi、SRAM 32KB / Flash 256KB）に**プログラム埋め込み**で配置
- 1パート分のデータを **C 配列**として定義（外部ファイル不要、SD カード不要）
- 解釈は楽器マイコン側で行い、CTRL パケットのテンポ・強弱を反映して NOTE パケット送出

## データ構造（C 言語表現）

```c
// 1音符の構造
typedef struct {
  uint8_t  pitch;     // MIDI ノート番号（0=休符として 0xFF を予約）
  uint16_t duration;  // 音価（ティック単位、下記参照）
} Note;

// パート全体
typedef struct {
  const char* part_name;   // "Soprano", "Alto", ...
  uint16_t    tick_per_beat; // 1拍あたりのティック数（通常 480）
  uint16_t    note_count;
  const Note* notes;
  uint16_t    entry_beat;  // 輪唱時の入りタイミング（拍番号）
} Score;
```

## ティックの定義

- **1 拍 = 480 tick**（MIDI 標準的慣行）
- これにより以下が整数で表現できる：
  - 4分音符 = 480
  - 8分音符 = 240
  - 16分音符 = 120
  - 3連符 = 160
  - 付点8分音符 = 360

## ピッチの表記

MIDI ノート番号で記述。よく使う音域：

| 音名 | MIDI 番号 |
|---|---|
| C4（中央ハ） | 60 |
| D4 | 62 |
| E4 | 64 |
| F4 | 65 |
| G4 | 67 |
| A4 | 69 |
| B4 | 71 |
| C5 | 72 |

休符は `pitch = 0xFF` で表現。

## サンプル：「かえるのうた」主旋律

```c
// パートA（Soprano、入り 拍 0）
const Note part_a_notes[] = {
  {60, 480}, {62, 480}, {64, 480}, {65, 480},  // ド レ ミ ファ
  {64, 480}, {62, 480}, {60, 480}, {0xFF, 480}, // ミ レ ド 休
  {64, 480}, {65, 480}, {67, 480}, {69, 480},  // ミ ファ ソ ラ
  {67, 480}, {65, 480}, {64, 480}, {0xFF, 480}, // ソ ファ ミ 休
  {60, 240}, {60, 240}, {60, 240}, {60, 240},  // ド ド ド ド
  {60, 240}, {60, 240}, {60, 240}, {60, 240},  // ド ド ド ド (16分)
  {60, 480}, {62, 480}, {64, 480}, {65, 480},  // ド レ ミ ファ
  {64, 480}, {62, 480}, {60, 480}, {0xFF, 480}, // ミ レ ド 休
};

const Score part_a = {
  .part_name = "PartA",
  .tick_per_beat = 480,
  .note_count = sizeof(part_a_notes) / sizeof(Note),
  .notes = part_a_notes,
  .entry_beat = 0,
};
```

輪唱では他パートの `entry_beat` を 8/16/24 拍とずらすことで輪唱効果を作る。

## 実装パターン

楽器マイコンのスケッチ構成例：

```
firmware/node_02/  (楽器1 = パートA)
├── platformio.ini
└── src/
    ├── main.cpp           // メインループ、CTRL/BEAT 受信、NOTE 送信
    ├── score.h            // Score 構造体定義
    ├── score_a.cpp        // パートA 楽譜データ
    └── network.cpp        // UDP 通信
```

## パート割り当て

| ノード | パート | 役割 |
|---|---|---|
| node_02 | A | 主旋律1（最初に入る） |
| node_03 | B | 主旋律2（2小節遅れて入る） |
| node_04 | C | 主旋律3（4小節遅れて入る） |
| node_05 | D | 主旋律4 or リズム（要検討） |

※ パート D をリズム（バッキング）にするか、4声輪唱の最後尾にするかは
課題曲決定後に確定。

## 拡張余地

- 曲を複数持たせて CTRL の `expression` フィールド等で曲選択
- 動的ロード（PC から SD/シリアル経由で楽譜転送）
