# 楽譜データの扱い（方針）

楽器マイコン側で「自分のパートの楽譜」をどう持たせるかの方針メモ。
具体的なデータ構造は実装フェーズで詰める。

## 方針

- 楽譜は **各楽器マイコン内部で保持**する（SD カードや外部ストレージに頼らない）
- パートごとに別の楽譜データを持たせ、4台で 4 声の輪唱を構成する
- 楽器マイコンは指揮者からの制御情報（テンポ・強弱など）を受けながら、
  楽譜に従って発音情報を PC に送る

## 未定（実装フェーズで詰める）

- データ形式（C 配列埋め込み／ヘッダファイル／独自フォーマット など）
- 音高・音価・休符の表現方法
- 輪唱の入りタイミングをどう指定するか
- 曲を複数持たせるかどうか、持たせる場合の切り替え方法
- 共通コードと楽譜データの分離（4 台でソースをどこまで共有するか）
