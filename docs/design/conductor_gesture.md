# 指揮ジェスチャ設計（方針）

指揮者マイコン内蔵の IMU から得たセンサ値を指揮コマンドに変換する。
決定経緯は [`../decisions/0003-conductor-imu.md`](../decisions/0003-conductor-imu.md) 参照。

## ハードウェア前提

- **MCU**: Arduino UNO R4 WiFi（本体内蔵 IMU を使用）
- 外付けセンサは使わない
- 装着方法（指揮棒・手の甲 など）は実機検証で決める

## 入力 → 出力 の対応（大枠のイメージ）

| 指揮の動き | 対応させたい演奏パラメータ |
|---|---|
| 振り下ろしのタイミング | 拍の同期トリガ |
| 振る速さ | テンポ |
| 振りの大きさ | 強弱 |
| 手先の細かい振動 | ビブラート |

## 未定（実装フェーズで詰める）

- サンプリング周波数・フィルタのカットオフなどの具体値
- 拍検出のアルゴリズム（どの軸・どの特徴量・どの条件で「拍」とするか）
- テンポ・強弱・ビブラートの具体的な推定方法と写像式
- 校正手順（個人差の吸収・起動時キャリブレーション）
- 誤認識時のフォールバック
