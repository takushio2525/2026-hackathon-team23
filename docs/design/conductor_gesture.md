# 指揮ジェスチャ設計（IMU → 指揮コマンド）

指揮者マイコン内蔵 IMU から得たセンサ値を、CTRL/BEAT パケットの各フィールドに
写像するアルゴリズムの設計。決定経緯は
[`../decisions/0003-conductor-imu.md`](../decisions/0003-conductor-imu.md) 参照。

## ハードウェア

- **MCU**: Arduino UNO R4 WiFi
- **IMU**: 本体内蔵 **LSM6DSOX**（3軸加速度＋3軸ジャイロ）
  - サンプリング周波数: 設定可能（本設計では **104 Hz** を採用）
  - 加速度レンジ: ±4 g
  - ジャイロレンジ: ±2000 dps
- **取り付け方法**: 指揮棒の根元に Arduino をマウント、または手の甲に固定
  （実機検証で最適な装着方法を決定）

## 座標系の定義

指揮者から見て：

- **X 軸**: 左右（右が正）
- **Y 軸**: 前後（前が正）
- **Z 軸**: 上下（上が正、重力加速度は -1g）

指揮の基本動作（4拍子）は X-Z 平面上の8字状軌跡を想定。

## 入力 → 出力 の写像

| 指揮動作 | 計測量 | 出力フィールド | 写像 |
|---|---|---|---|
| 振り下ろしのタイミング | Z 軸加速度の正→負ゼロクロス | **BEAT パケット送出** | 拍頭検出 |
| 振る速度（テンポ） | 拍と拍の時間間隔 | `tempo_bpm_x10` | 60000 / interval[ms] × 10 |
| 振幅の大きさ | 1拍内の加速度 RMS（または軌跡の長さ） | `dynamics` | 線形写像 0〜127 |
| 手先の細かい振動 | 高周波成分のエネルギー（HPF後の RMS） | `vibrato_depth` | 線形写像 0〜127 |
| 振動の周波数 | 高周波成分のピーク周波数（FFT or ZCR） | `vibrato_rate_x10` | 0〜200（0〜20 Hz） |

## 信号処理パイプライン

```
[IMU 生値 104Hz]
    │
    ├─→ [ローパス 10Hz] ──→ 拍検出（Z軸ゼロクロス）─→ BEAT
    │                            └→ テンポ推定 ─→ tempo_bpm
    │
    ├─→ [拍区間 RMS] ─→ ダイナミクス推定 ─→ dynamics
    │
    └─→ [ハイパス 8Hz] ─→ 包絡線 ─→ vibrato_depth
                            └→ ピーク周波数 ─→ vibrato_rate
```

### 1. 前処理

- 重力成分の除去（静止時の Z = -1g を引く）
- 移動平均または IIR ローパス（カットオフ 10 Hz）でノイズ除去

### 2. 拍検出

- Z 軸ローパス後の信号で **加速度ゼロクロス**（負→正）を拍頭とする
- ヒステリシスを設けてダブルカウント防止
- 拍頭検出時に BEAT パケットを即送出

### 3. テンポ推定

```
interval = current_beat_time - last_beat_time   // ms
tempo_bpm = 60000 / interval
```

- 指数移動平均（EMA, α=0.3）で平滑化
- 範囲外（< 40 BPM, > 240 BPM）は前回値を保持

### 4. ダイナミクス推定

- 1 拍区間（最新の拍 → 前の拍）の加速度 RMS を計算
- 校正値（小→大の指揮振幅サンプルで取った 0/127 の境界）で線形写像

### 5. ビブラート検出

- ハイパス（カットオフ 8 Hz）で高周波成分を抽出
- 包絡線（絶対値→ローパス）の RMS → `vibrato_depth`
- 包絡線のゼロクロス周波数 or 短時間 FFT のピーク → `vibrato_rate`

## 校正手順（実装フェーズで実施）

1. **静止校正**: 数秒静止して重力ベクトルを取得
2. **動作校正**: 「弱く振る／普通に振る／強く振る」を各5秒記録し、
   `dynamics` の 0/64/127 に対応する加速度 RMS を決定
3. **ビブラート校正**: 手先を意図的に振動させて閾値を決定

校正値はフラッシュ（EEPROM 相当領域）に保存し、起動時に読み込む。

## 実装ステップ

| Step | 内容 | 検証方法 |
|---|---|---|
| 1 | IMU 値を Serial に dump | プロット確認 |
| 2 | 拍検出ロジック実装 | LED 点滅で確認、メトロノームと比較 |
| 3 | テンポ推定 + CTRL 送出 | 別マイコンで受信して LED 点滅 |
| 4 | ダイナミクス推定 | dynamics 値を Serial に dump |
| 5 | ビブラート検出 | 同上 |
| 6 | 校正手順実装 | 校正→演奏で挙動確認 |

## 既知のリスクと対策

| リスク | 対策 |
|---|---|
| 拍検出の誤検出（手の震え等） | ヒステリシス＋最小拍間隔（200 ms = 300 BPM 上限） |
| テンポの暴れ | EMA 平滑化＋範囲外フィルタ |
| 校正値の個人差 | 起動時に短い校正フェーズを設ける |
| Arduino UNO R4 の処理負荷 | サンプリング 104 Hz 程度なら余裕、FFT は使わず ZCR で代用可能 |
