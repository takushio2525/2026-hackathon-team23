## モード

| モード | 内容 |
|---|---|
| 自由演奏 | 実際に振ったBPMで56拍サイクルを繰り返す |
| ゲーム | 目標100 BPMを記憶し、56拍のテンポ維持を0〜100点で採点する |

メニューでは左右振りでカーソルを移動し、縦振りで決定します。重力ベクトルを基準に縦・横を判定するため、
センサーの取り付け角度に依存しにくい設計です。

## ガイド

| 区間 | ガイド強度 |
|---|---|
| 0〜15拍 | 100% |
| 16〜31拍 | 100%から0%へ線形に減衰 |
| 32〜55拍 | ガイドなし |

指揮者LEDとPCのクリック音が同じ強度スケジュールを使います。

## 採点

各拍について、実際の拍間隔と100 BPMの目標間隔との差を求めます。
ガイドが弱い区間ほど重い重みを付け、加重平均誤差を0〜100点へ写像します。
目標間隔の50%ずれを0点相当とします。

56拍に到達すると`Result`へ遷移して得点を固定し、縦振りでメニューへ戻ります。

## PC表示

node_02がCTRLをUIパケットへ中継します。Processingは`state`と`mode`から
待機、メニュー、自由演奏、ゲーム、結果の画面を自動選択します。UIが2秒途絶えると
指揮者のリセットと判断し、発音を止めて待機画面へ戻ります。
