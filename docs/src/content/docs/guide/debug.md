title: デバッグする
description: LED、シリアルログ、画面から不具合箇所を切り分ける
---

## 切り分け順

1. 指揮者のLEDで状態を確認する
2. 楽器が`OrchestraAP`へ接続できているか確認する
3. BEATの`beatNo`が増えているか確認する
4. 楽器のNOTEがUSBへ出ているか確認する
5. Processingが正しいポートと音色を開いているか確認する

## ログモード

`platformio.ini`の`build_flags`で`-DSERIAL_DEBUG=1`にすると診断ログを出せます。
通常演奏へ戻すときは`0`へ戻し、再ビルドしてください。

## 重要なフィールド

| フィールド | 確認内容 |
|---|---|
| `beatNo` | 欠番や停止がないか |
| `bpm` | 振りの変更へ追従するか |
| `offsetMs` | 大きく発散していないか |
| `playAtMasterMs` | 未来時刻になっているか |
| `partId` | 接続したノードと一致するか |
| `instrumentId` | 期待する音色番号か |

## Processing側

ウィンドウ下部のノード表示、受信数、直近イベント、UI同期表示を確認します。
UIが2秒止まると待機へ戻るため、音声だけでなく指揮者・node_02間も調べてください。
