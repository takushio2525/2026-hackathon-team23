## プロジェクト題目

**タクトーン — IMU ジェスチャーで奏でる Arduino オーケストラ**

（第 4 回ミーティング 2026-05-13 の議題 2-1「題名決め」で確定。
メインタイトル「タクトーン」＋サブタイトル「IMU ジェスチャーで奏でる Arduino オーケストラ」）

## 一言でいうと

**Arduino 系マイコン（現行 test_v2 は 4 台、production 想定は 5 台）＋ PC で、
指揮者の振りに合わせて演奏する電子オーケストラ**。

- 1 台が **指揮者**: 外付け IMU で腕の振りをセンシングする
- **現行 test_v2 は楽器 3 台、production 想定は楽器 4 台**: それぞれ別のパートを担当して、輪唱を演奏する
- **PC（Processing）** が、楽器マイコンから受け取った発音情報を音色合成してスピーカから鳴らす

指揮者が腕を振る速さでテンポが変わり、振りの大きさで音量が変わる。
**音楽未経験の人がいきなり指揮者を任されても演奏できる** ことを目指している。

## なぜ作るのか

千葉工業大学 情報変革科学部 情報工学科の **ハッカソン課題** として与えられた制約:

- 人数分 − 1 の Arduino を使う（6 名チーム → 5 台が production 想定。現行 test_v2 は 4 台で稼働中）
- 各楽器で異なるパートを演奏する
- 主旋律 3 + リズム 1 以上、輪唱曲を演奏する
- 同期手法に独創性が欲しい

この制約の中で「他のチームと差別化できる、見ていて楽しい同期手法は何か」を議論し、
**IMU ジェスチャ同期** に落ち着いた（[ADR-0003](/decisions/0003-conductor-imu/) の決定経緯参照）。

第 4 回ミーティングで確定した目的の言い回し（議事録 9-2 より要約）:

> 時間帯や天候、ネットワーク環境などの外部条件に左右されず、明るさなどの制約も
> 受けない環境で、音楽経験のない人でも **直感的な指揮動作だけで演奏に参加できる**
> システムを開発する。さらに加速度の大きさに応じた音量変化や LED による視覚演出を
> 加え、誰でも簡単に楽しめる体験型電子オーケストラを目指す。

詳しくは [なぜ作るのか](/concept/why/) を読んでください。

## 評価軸（授業資料より）

| 評価項目 | 内容 | 本チームの狙い |
|---|---|---|
| 同期信号の受信精度 | 正しく受信できているか | UDP + 独自プロトコルで欠損時も復帰 |
| タイミング誤差 | 演奏ズレの短さ | **楽器間 ≤ 20 ms** を目標（[ADR-0006](/decisions/0006-sync-error-moe-20ms/)） |
| エンターテインメント性 | 見た目の楽しさ・華やかさ | 指揮ジェスチャで「指揮者感」を演出 |
| 楽器音らしさ | 目的楽器に聞こえるか | Processing で金管の倍音構造 + ADSR |

詳細根拠: `references/lectures/HCK1_01_Kickoff_A.pdf`（p.65）

## 何が面白いか（独創性のポイント）

### 1. IMU だけで「指揮者感」を出す

カメラ画像認識ではなく、**ボード内蔵 IMU の加速度ノルム** だけで拍とテンポを検出する。
外部機材ゼロで、振りの方向に依存しない（縦でも横でも斜めでも拾える）。

詳しくは [シナリオと体験](/concept/scenario/) と [同期戦略](/architecture/sync/) を参照。

### 2. 楽譜内蔵 + 拍番号同期で「途中起動」に強い

楽譜は楽器マイコン内部に持つ。曲全体の長さで mod 演算するので、**PC を曲の途中で起動しても
「今の拍」から鳴り始める**。デモ中にやり直しても止まらない。

詳しくは [楽譜フォーマット](/architecture/score/) を参照。

### 3. 楽器番号付き NOTE で音色を切り替える

NOTE パケットに `instrumentId` を載せ、PC 側 `pc_app/test_v2/orchestra_resynth/data/` 配下の
JSON をファイル名昇順で配列化し、`instrumentId` を index として参照する。
**`pc_app/.../data/` に JSON を追加するだけで楽器のラインナップを増やせる**
（`sound_lab/` で試作 → コピー反映）。

詳しくは [通信プロトコル](/architecture/protocol/) と
[pc_app の歩き方](/code/pc-app/) を参照。

## 現状（2026-05-14 時点）

- **`firmware/test_v2/` の輪唱版** で「きらきら星」3 声輪唱が動作
- 同期誤差は実測 ≤ 20 ms（楽器間ピア・ピア比較）
- production 系は素テンプレ（本実装は test_v2 → production に取り込む運用）
- 第 4 回ミーティング（2026-05-13）で **プロジェクト計画書 / システム設計書** を確定
- **来週 2026-05-20 のプレゼンテーション** に向け、スライド作成中
- 成果発表会は **2026-07-01**

## 制約と未決事項

- ハードウェア:
  - 指揮者は **XIAO ESP32-S3 Sense + GY-521（MPU6050）** を採用
    （Arduino UNO R4 WiFi の内蔵 IMU はテスト時の性能不足で外付け IMU に切替）
  - 楽器は **Arduino UNO R4 WiFi**
- 各楽器は **異なるパート** を演奏する（同一パート複数台は NG）
- はんだ付けは各自で実施。無い機材はチームで相談の上購入
- production 想定の **5 番目のノード（node_05）** は本実装未着手（ドラム想定）。現行 test_v2 は `node_01〜04` の 4 台で稼働中
