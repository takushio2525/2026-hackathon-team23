---
title: 時刻同期アルゴリズム
description: 指揮者時計を各楽器のローカル時計へ写す
---

## オフセット

楽器が時刻`localReceiveMs`に、指揮者時刻`timestampMs`のパケットを受け取った場合、
観測値を次で作ります。

```text
offsetSample = timestampMs - localReceiveMs
masterNow ≈ localNow + offsetMs
```

直近2秒の観測窓で、もっとも大きい`offsetSample`（= 配送遅延がもっとも小さいサンプル）を採用します。Wi-Fiでは遅れて届く信号が多いため、平均を取るよりも「早く届いた信号」を基準にした方が、予定時刻を正しくローカル時計へ写せます。

## 予約時刻変換

```text
targetLocalMs = playAtMasterMs - offsetMs
```

楽器の2 msループでこの時刻を過ぎたらNOTEを送ります。受信時点ですでに期限を過ぎていても、
音を捨てず即時発火します。

## リセット追従

指揮者の再起動でオフセット観測が1000 ms以上飛んだ場合、観測窓を待たず即時採用します。
これによりマスタ時計の巻き戻りから1パケットで復旧します。

## 重複排除

BEATは4連送されますが、同じ`beatNo`は1つの発音予約として扱います。重複は時計同期の観測には使いますが、後着の重複は最大値を更新しないため、発音時刻を遅らせません。
