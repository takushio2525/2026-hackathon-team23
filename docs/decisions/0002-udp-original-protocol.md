# ADR-0002: 通信方式は UDP ベースを採用

**ステータス**: Accepted
**決定日**: 2026-04-15（第1回ミーティング）
**関連議事録**: [`meetings/23_第1回議事録_24G1131-1776302030881.pdf`](../../meetings/23_第1回議事録_24G1131-1776302030881.pdf)

## 背景

指揮者マイコンから楽器マイコンへ、演奏制御情報をほぼリアルタイムで届ける
必要がある。さらに楽器マイコンから PC（Processing）へ発音情報を流す構成も
前提となる。

Arduino UNO R4 WiFi の標準機能で扱える低遅延・1対多送信が向く方式として、
UDP を軸に検討した。

## 決定事項

通信方式は **UDP ベース** とする。独自プロトコルを設計する前提で進め、
同期誤差が許容範囲を超えた場合は再検討する。

具体的なパケット構造・同期手順・ポート構成などは、実装フェーズで
[`../design/protocol.md`](../design/protocol.md) に落とす。

## 理由

- Arduino UNO R4 WiFi の `WiFiUDP` ライブラリが標準で使え、1対多の配信に向く
- 評価軸「演奏タイミングの誤差」に対して低遅延の UDP が有利
- パケット内容を自分たちで設計できるため、同期や表現（ビブラート等）を
  自由に乗せられる

## 結果・影響

- **ポジティブ**: 低遅延・拡張性・実装量が読みやすい
- **ネガティブ / リスク**: パケロス・ジッタ対策は自分たちで設計する必要がある

## 再検討条件

- 実機測定で楽器間の同期誤差が人間の可聴閾値を超えて改善できない場合
- パケロス率が高止まりしてフォールバックでも救えない場合
