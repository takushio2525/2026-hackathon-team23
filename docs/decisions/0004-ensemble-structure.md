# ADR-0004: 5台構成（指揮1＋楽器4）・金管四重奏・輪唱曲

**ステータス**: Accepted
**決定日**: 2026-04-15（第1回ミーティング）
**関連議事録**: [`meetings/23_第1回議事録_24G1131-1776302030881.pdf`](../../meetings/23_第1回議事録_24G1131-1776302030881.pdf)

## 背景

授業資料の制限（[Kickoff p.66](../../references/lectures/HCK1_01_Kickoff_A.pdf)）：

- **人数分 − 1 の Arduino を使用する**（本チームは6名 → 5台）
- **各楽器で楽曲の異なるパートを演奏する**

課題曲（[Kickoff p.64](../../references/lectures/HCK1_01_Kickoff_A.pdf)）：

- 最低でも主旋律3つ、リズム1つ
- **輪唱曲**を演奏する（森のクマさん・静かな湖畔・かえるのうた 等）

## 決定事項

以下の構成で進める：

| 項目 | 決定内容 |
|---|---|
| ハードウェア台数 | Arduino UNO R4 WiFi × **5台** |
| 役割分配 | **指揮者 × 1 + 楽器 × 4** |
| 楽器の種類 | **金管楽器**（トランペット／ホルン／トロンボーン／チューバ等を想定） |
| 曲の種類 | **輪唱曲**（主旋律4パートで輪唱構造、リズムは金管内で兼ねる or 別途検討） |
| 音声出力 | **Processing（PC）側**で音色合成・再生（Arduino は音符情報送出のみ） |

## 理由

- 授業制約「人数分 − 1」に完全準拠
- 金管楽器4パートは輪唱曲（4声輪唱）と相性が良い
- 金管楽器の倍音構造は比較的シンプルで、Processing での音色合成が
  他の楽器群（弦・鍵盤等）より実装しやすい
- 音色生成を Processing 側に寄せる構成は授業資料の前提と一致

## 未決事項（本 ADR の範囲外）

- **具体的な課題曲の選定**（森のクマさん／静かな湖畔／かえるのうた 等、次回決定）
- 4声の役割分担（どのマイコンがどの声部か）
- リズム要素をどう扱うか（金管パート内で刻むか、Processing 側で別生成するか）
- エンターテインメント要素（LED・可動装置等）の有無

## 結果・影響

- **ポジティブ**: 構成がシンプルで責務分離が明快
- **ネガティブ / リスク**:
  - 金管楽器音色の合成クオリティが評価軸「楽器音らしさ」に直結
  - Processing 側の実装負荷が高く、担当（梅澤）のタスク密度が上がる
- **次のアクション**:
  - [`docs/design/score_format.md`](../design/score_format.md) で楽譜データ形式を設計
  - [`docs/roles.md`](../roles.md) で4楽器の担当割り当てを明示（未決項目として管理）
