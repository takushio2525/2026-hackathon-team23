# ADR-0004: 5台構成（指揮1＋楽器4）・金管四重奏・輪唱曲

**ステータス**: Accepted
**決定日**: 2026-04-15（第1回ミーティング）
**関連議事録**: [`meetings/0415_1回/23_第1回議事録_24G1131-1776302030881.pdf`](../../meetings/0415_1回/23_第1回議事録_24G1131-1776302030881.pdf) ／ AI要約 [`meetings/0415_1回/第1回議事録_AI要約byClaude.md`](../../meetings/0415_1回/第1回議事録_AI要約byClaude.md)
**補足**: 第2回ミーティング（2026-04-22）で「指揮棒 + 楽器」の構成を正式に再確認（[第2回議事録PDF](../../meetings/0422_2回/23_第2回議事録_24G1075.pdf) ／ [AI要約](../../meetings/0422_2回/第2回議事録_AI要約byClaude.md)）

## 背景

授業資料の制約：

- 人数分 − 1 の Arduino を使用する（本チームは6名 → 5台）
- 各楽器で異なるパートを演奏する
- 最低でも主旋律3 + リズム1、輪唱曲を演奏する

## 決定事項

| 項目 | 決定内容 |
|---|---|
| ハードウェア台数 | Arduino UNO R4 WiFi × **5台** |
| 役割分配 | **指揮者 × 1 + 楽器 × 4** |
| 楽器の種類 | **金管楽器** を想定 |
| 曲の種類 | **輪唱曲**（候補は次回決定） |
| 音声出力 | **Processing（PC）側**で音色合成・再生 |

## 理由

- 授業制約「人数分 − 1」に準拠
- 金管楽器4パートと輪唱（多声）の相性が良い
- 金管の倍音構造は他楽器群より Processing での合成が扱いやすい
- 音色生成を PC 側に寄せる構成は授業資料の前提と一致

## 未決事項（本 ADR の範囲外）

- 具体的な課題曲の選定
- 4声の具体的な役割分担（どのマイコンがどの声部か）
- リズム要素の扱い
- エンタメ要素（LED・可動装置等）の有無

## 結果・影響

- **ポジティブ**: 構成がシンプルで責務分離が明快
- **ネガティブ / リスク**:
  - 金管音色の合成クオリティが評価軸「楽器音らしさ」に直結
  - Processing 担当（梅澤）のタスク密度が上がる
