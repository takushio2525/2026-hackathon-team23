# ADR-0003: 指揮者マイコンに IMU を搭載しジェスチャで演奏を制御

**ステータス**: Accepted
**決定日**: 2026-04-15（第1回ミーティング。議事録 PDF には未記載・本 ADR で補完）
**関連議事録**: [`meetings/23_第1回議事録_24G1131-1776302030881.pdf`](../../meetings/23_第1回議事録_24G1131-1776302030881.pdf)

## 背景

授業資料（[Kickoff p.61 "同期の取り方"](../../references/lectures/HCK1_01_Kickoff_A.pdf)）で、
同期の取り方によってチームの独創性を発揮することが明示されている。提示された例：

- LED の点滅に合わせて同期
- リズム用楽器の音で同期
- **指揮者人形のジェスチャに合わせて同期**

また第1回ミーティングでは以下が決まっていた：

- 指揮者 Arduino は塩澤が担当
- 指揮によってテンポが可変になる（倍速対応）
- 余裕があれば強弱・ビブラートにも着手
- 「ここしかない独創性がある」を狙う

## 決定事項

指揮者マイコン（Arduino UNO R4 WiFi）に **IMU（加速度・角速度センサ）** を搭載し、
**指揮棒のジェスチャを認識**して、演奏制御コマンドを楽器マイコンへ配信する。

## 理由

- 授業例示の「ジェスチャ同期」に直結する独創性
- 単なる LED 点滅同期より表現の幅が広い：
  - **振幅の大きさ** → 強弱（ベロシティ）
  - **振る速度** → テンポ
  - **手先の細かい振動** → ビブラート
  - **振り下ろしのタイミング** → 拍の同期トリガ
- UNO R4 WiFi の本体に IMU（LSM6DSOX）が搭載されているため、外付け回路不要で
  実装コストが低い
- 評価軸の「エンターテインメント性（見た目の楽しさ・華やかさ）」にも寄与

## 代替案と比較

| 案 | 採否 | 理由 |
|---|---|---|
| IMU ジェスチャ | ✅ 採用 | 独創性・表現幅・実装容易（本体内蔵） |
| LED 点滅 + 光センサ | ❌ 不採用 | 独創性が弱い・環境光に左右される |
| リズム楽器音で同期 | ❌ 不採用 | ドラム担当が別途必要、本システムの金管構成と合わない |
| ボタン＋つまみの物理UI | ❌ 不採用 | 「指揮者」の雰囲気が出ない |

## 結果・影響

- **ポジティブ**: 独創性評価・エンタメ性評価ともに上振れ要素
- **ネガティブ / リスク**:
  - ジェスチャ認識アルゴリズムの実装コスト
  - 誤認識による演奏破綻のリスク → 閾値調整とフィルタ設計が必須
  - IMU データをローカル処理するか PC に投げるかで設計が変わる（別途検討）
- **次のアクション**:
  [`docs/design/conductor_gesture.md`](../design/conductor_gesture.md) に
  IMU 値から指揮コマンドへの写像アルゴリズムを設計する

## 再検討条件

- ジェスチャ誤認識率が実用水準に収まらない場合、補助入力（ボタン等）を併用
